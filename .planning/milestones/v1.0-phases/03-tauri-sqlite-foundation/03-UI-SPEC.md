## Interaction Contract

### Splash Screen

- Displayed full-screen at app launch before `<RouterView>` mounts.
- Shows while `useDatabase().initialize()` is in progress (typically 50–200ms).
- Transitions: `opacity 0` → `opacity 1` on mount (200ms ease-in); `opacity 1` → `opacity 0` on dismiss (300ms ease-out) then `display: none`.
- Loading indicator: CSS border-spinner, 32px, `animate-spin`.
- Text cycles: "Initializing database..." → "Running migrations..." → "Ready." (300ms hold) → dismiss.
- On error: spinner stops; error card appears in place of loading text; no auto-dismiss.

### Dashboard Navigation Tiles

- Two tiles in a responsive grid: 1 column on narrow window, 2 columns at `sm` breakpoint (640px+).
- Combat Tracker tile: `<RouterLink to="/combat">` — fully clickable card.
- Sync Data tile: `<div>` (not a link), `cursor-default`, `opacity-60`, "Coming soon" annotation.
- Tile hover (Combat Tracker only): `shadow-md` → `shadow-lg`, transition 150ms.
- No active state animation — tile click navigates immediately via vue-router.

### Router

- Routes: `/` → Dashboard, `/combat` → CombatTracker.
- App shell: `<RouterView>` replaces direct `<CombatTracker />` in `App.vue`.
- No persistent nav bar in this phase — Combat Tracker keeps its own header unchanged.
- Back navigation from Combat Tracker: not required in this phase (DM navigates via window controls or keyboard).

### Loading States

| State | Visual |
|-------|--------|
| DB initializing | Splash screen visible, spinner active |
| Migrations running | Splash screen visible, spinner active, text updated |
| Ready | Splash "Ready." text 300ms, then fade out to Dashboard |
| Init error | Spinner hidden, error card visible, "Retry Initialization" button active |
