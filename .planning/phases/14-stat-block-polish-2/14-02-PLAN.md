# Plan 14-02: Component Rendering — Colors, DC, Structured Damage, Layout

## Objective

Apply all visual changes to `CreatureStatBlock.tsx` using the data types established in Plan 14-01. This covers: per-stat color classes on `StatItem`, inline DC display for saves/perception, Spell DC / Class DC second row, structured damage rendering with `text-pf-blood` type coloring, weapon group badge, `additionalDamage` entries, Skills section as `<Collapsible>`, gradient section headers across all three sections, updated ability trait colors, and ability body text opacity.

All color classes, class strings, and copy come directly from the approved `14-UI-SPEC.md`.

## must_haves

- `StatItem` renders `"Fort +10 (DC 20)"` — derived DC inline for saves and perception
- AC renders with `text-pf-gold`, saves with `text-pf-threat-low`, perception with `text-pf-gold-dim`
- Spell DC / Class DC second row appears below main stats row when values are present
- Section headers for Strikes, Abilities, AND Skills have gradient + left-border treatment
- Skills section is a `<Collapsible defaultOpen>` matching Strikes/Abilities structure
- Ability traits render with `bg-primary/10 text-primary border border-primary/20`
- Ability body text uses `text-foreground/80`
- Strike damage renders each `{ formula, type }` entry; type is wrapped in `text-pf-blood`
- Group badge renders below strike name when `strike.group` is present
- `additionalDamage` entries render as separate lines with label and colored type

```tsx
function StatItem({ label, value, modifier, highlight, colorClass, showDc }: StatItemProps) {
  const displayValue = modifier && value > 0 ? `+${value}` : `${value}`
  const dc = showDc ? ` (DC ${10 + value})` : ''
  return (
    <div className="text-center">
      <p className="text-xs text-muted-foreground mb-1">{label}</p>
      <p
        className={cn(
          "font-mono font-bold text-lg",
          highlight && "text-pf-threat-extreme",
          colorClass
        )}
      >
        {displayValue}{dc}
      </p>
    </div>
  )
}
```

**3. Update the six `StatItem` calls in the core stats grid** (lines 47–55 region):

```tsx
<div className="grid grid-cols-6 gap-4">
  <StatItem label="HP" value={creature.hp} highlight />
  <StatItem label="AC" value={creature.ac} colorClass="text-pf-gold" />
  <StatItem label="Fort" value={creature.fort} modifier colorClass="text-pf-threat-low" showDc />
  <StatItem label="Ref" value={creature.ref} modifier colorClass="text-pf-threat-low" showDc />
  <StatItem label="Will" value={creature.will} modifier colorClass="text-pf-threat-low" showDc />
  <StatItem label="Perception" value={creature.perception} modifier colorClass="text-pf-gold-dim" showDc />
</div>
```

Note: `highlight` on HP retains `text-pf-threat-extreme` (already correct). The `cn()` call applies `colorClass` when provided, `text-pf-threat-extreme` when `highlight` is true — if both present, both classes apply (HP stays red since `highlight` is only set for HP). AC has no `modifier` prop so it renders plain number without `+` prefix.
