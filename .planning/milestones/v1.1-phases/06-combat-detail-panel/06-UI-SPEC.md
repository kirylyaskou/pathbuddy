## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chip padding-x |
| sm | 8px | Compact element spacing, button padding-x in tight rows |
| md | 16px | Default section padding, card padding-x |
| lg | 24px | Section padding in stat block body, filter area padding |
| xl | 32px | Layout column gaps, major section breaks |
| 2xl | 48px | Page-level vertical spacing |
| 3xl | 64px | Reserved for outer layout shell (not used in detail panels) |

Exceptions:
- Trait pill padding: 4px vertical / 8px horizontal (xs/sm)
- Section divider bar height: 28px (7 in Tailwind, closest to 8px grid via py-2)
- Level badge: min-width 28px, height 24px — intentional small badge, not touch target
- Entity row height: 40px (`:item-size=40` established in VList — do not change)
- Stats header row: py-3 (12px) to maintain density in the overlay block

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | body (system-ui) | Ability descriptions, section content, filter labels |
| Label | 12px (text-xs) | 400 (normal) | 1.4 | body (system-ui) | Stat labels above values ("HP", "AC", "Fort"), filter control labels, condition badge text |
| Heading | 16px (text-base) | 700 (bold) | 1.2 | display (Cinzel) | Item names within stat block sections, creature name in overlay |
| Display | 20px (text-xl) | 700 (bold) | 1.2 | display (Cinzel) | Entity name in StatBlock header (Row 1), panel/section headings |

Weight constraint: exactly 2 weights — 400 (normal) and 700 (bold). No intermediate weights.

Section divider headings: 12px (text-xs), 700 bold, uppercase, tracking-wider — matches existing `CreatureDetailPanel` section divider pattern.

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | charcoal-800 | #1a1a1a | Main app background, panel body |
| Secondary (30%) | charcoal-600 / charcoal-900 | #2a2a2a / #141414 | Panel surfaces, section divider bars, stats header row, filter area |
| Accent (10%) | gold (DEFAULT) | #c9a84c | Reserved for specific elements only — see list below |
| Destructive | crimson-light | #b03030 | Error states, resolve-error messages |

Accent (`text-gold`, `border-gold`, `bg-gold`) reserved ONLY for:
- Entity name in StatBlock header (Row 1 text color: `text-gold`)
- Selected combat tracker row indicator: `border-l-4 border-gold` + `bg-charcoal-600` (elevated)
- Currently-active turn badge in CombatTracker (established: `bg-gold text-charcoal-950`)
- Canonical cross-reference link icon on hover (`hover:text-gold`)
- Focus ring on form inputs (`focus:ring-gold`, `focus:border-gold`)
- Tier badge for "Elite" label: gold background pill — `bg-gold text-charcoal-950`

Rarity pill colors (trait pills in StatBlock header):
- Common: `bg-stone-600 text-stone-200` (stone/neutral)
- Uncommon: `bg-amber-700 text-amber-100`
- Rare: `bg-blue-700 text-blue-100`
- Unique: `bg-purple-700 text-purple-100`

Level badge colors in entity list rows (relative to `partyLevel`):
- creature level <= partyLevel - 3: `bg-stone-600 text-stone-200` (gray — trivial/lackey)
- creature level = partyLevel - 2 or partyLevel - 1: `bg-emerald-800 text-emerald-200` (green — weak/moderate)
- creature level = partyLevel or partyLevel + 1: `bg-charcoal-500 text-stone-200` (neutral — standard/strong)
- creature level = partyLevel + 2: `bg-amber-700 text-amber-100` (amber — boss)
- creature level >= partyLevel + 3: `bg-crimson text-stone-100` (crimson — severe/solo boss)

Tier badge for "Weak" label: `bg-charcoal-500 text-stone-400` (muted, no color emphasis).
Tier badge for "Normal": not shown.

Entity type icon: `text-stone-400` (muted, not accent). Caster icon same.

Ongoing damage value: `text-crimson-light`.
Regeneration ON: `text-stone-400` (neutral). Regen OFF: `bg-crimson text-stone-100` (existing pattern in CreatureCard).
Fast healing value: `text-emerald-400`.

## Interaction Contracts

### Row Click — Combat Tracker
- Target: entire CreatureCard row (the wrapping `div`) is clickable, not just the "View Stat Block" button
- Pointer cursor: `cursor-pointer` on row wrapper
- Selection is not a toggle — clicking an already-selected row keeps it selected
- On click: `emit('select', creature.id)` from CombatTracker to CombatView
- CombatView sets `selectedCombatantId.value = creatureId`
- CombatDetailPanel receives updated prop and fetches rawData from DB by `creature.sourceId`
- If `creature.sourceId` is null (manually added creature): LiveCombatOverlay renders normally; StatBlock renders the "No compendium entry found" fallback copy

### Row Click — Compendium Entity List
- Target: entire row in CreatureBrowser is clickable (existing behavior in compendium mode)
- On click: CompendiumView updates its `selectedEntity` ref; StatBlock in right column renders with new rawData
- Right column scrolls to top on entity change (use `scrollTop = 0` via ref or `scrollIntoView`)

### Canonical Cross-Reference Navigation (within StatBlock)
- Clicking the ↗ icon on a non-unique item: replaces current StatBlock content with the canonical entity's data
- Back button (← Back) appears in StatBlock header when navigation history length > 0
- Navigation history is local to the StatBlock instance; it does not affect CombatView or CompendiumView selection state

### Infinite Scroll (Entity List)
- VList sentinel div positioned as last item in list
- When sentinel enters viewport (IntersectionObserver), load next page (offset += 200)
- During page load: spinner renders below last visible item
- When all results loaded and total < current offset + 200: sentinel removed, no more pages

### Stat Block Scroll Reset
- When `selectedCombatantId` or `selectedEntity` changes: scroll the stat block panel to top via `nextTick` + `scrollTo(0, 0)` on the panel ref — matching the existing pattern in CreatureDetailPanel.vue
