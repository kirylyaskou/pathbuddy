## Markup Contracts

### 1. Skills Section — CreatureStatBlock.tsx

**Current behavior (lines 202–217):** Only renders skills present in Foundry data; hidden entirely if `creature.skills.length === 0`.

**Required behavior:** Always render the skills section with all 17 skills. The `creature.skills[]` array from the mapper will always have 17+ entries (17 standard skills + any Lore skills). Untrained derived skills have `calculated: true` on the skill object.

**Visual distinction for untrained skills (D-08 — Claude's discretion):** Apply `opacity-40` to the entire skill `<span>` when `skill.calculated` is true. This communicates "derived, not explicitly rated" without introducing new color tokens or text decorations.

**Complete replacement for the Skills section block:**

```tsx
{/* Skills — always rendered, all 17 standard skills */}
<div className="p-4">
  <h4 className="font-semibold mb-2">Skills</h4>
  <div className="flex flex-wrap gap-x-4 gap-y-1 text-sm">
    {creature.skills.map((skill) => (
      <span key={skill.name} className={skill.calculated ? "opacity-40" : ""}>
        <span className="text-muted-foreground">{skill.name}</span>{" "}
        <span className="font-mono text-primary">
          {skill.modifier >= 0 ? "+" : ""}{skill.modifier}
        </span>
      </span>
    ))}
  </div>
</div>
<Separator />
```

Notes:
- The `{creature.skills.length > 0 && (...)}` conditional wrapper is **removed** — skills section is always present.
- Trained skills (explicit in Foundry data): full opacity, `text-muted-foreground` name + `text-primary` modifier.
- Untrained calculated skills: `opacity-40` wrapper — both name and modifier fade together. No `~` prefix.
- Modifier sign is explicit: `+` for non-negative values. Negative values render naturally (e.g., `-1`).
- `<Separator />` after skills is retained — matches every other stat block section.

### 2. Source Filter — BestiaryFilterBar.tsx

**Current behavior (lines 27–31, 103–120):** `sources` is `string[]`; each string is used as both `value` and display text (folder name like `"pf2e"`).

**Required behavior:** `sources` is `{ pack: string; name: string }[]`. The `SelectItem` `value` is `source.pack` (unchanged filter logic); display text is `source.name` (human-readable book title like `"Player Core"`).

**State type change:**

```tsx
// BEFORE:
const [sources, setSources] = useState<string[]>([])

// AFTER:
const [sources, setSources] = useState<{ pack: string; name: string }[]>([])
```

**SelectItem rendering change:**

```tsx
// BEFORE:
{sources.map((s) => (
  <SelectItem key={s} value={s}>
    {s}
  </SelectItem>
))}

// AFTER:
{sources.map((s) => (
  <SelectItem key={s.pack} value={s.pack}>
    {s.name}
  </SelectItem>
))}
```

**No other changes to BestiaryFilterBar.tsx.** The `filters.source` value remains a `source_pack` string. `setFilter('source', v === '__all__' ? null : v)` is unchanged — `v` is still the pack string.
