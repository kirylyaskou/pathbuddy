## Objective

Wire the `ac` field end-to-end through the Characters → Combat integration path:
1. Fix `handleAddToCombat` to parse `rawJson` as `PathbuilderBuild` (not `PathbuilderExport`),
   compute AC from `acTotal`, and include it in the created `Combatant`.
2. Update `loadEncounterIntoCombat` in `encounter-persistence.ts` to restore `ac` from the
   snapshot when rebuilding `Combatant` objects.

Depends on **45-01** (both `Combatant.ac?: number` and `EncounterCombatantRow.ac?: number` must exist).

## Context

- **rawJson bug**: `CharactersPage.tsx` currently parses `character.rawJson` as `PathbuilderExport`
  (`{ success, build }`) and passes `exp.build` to `calculatePCMaxHP`. This is wrong — `upsertCharacter`
  in `characters.ts` stores `JSON.stringify(build)` directly (the `PathbuilderBuild`, not the wrapper).
  `exp.build` is therefore `undefined` at runtime. Fix: parse as `PathbuilderBuild` directly.
- **AC formula**: `ac = build.acTotal.acProfBonus + build.acTotal.acAbilityBonus + build.acTotal.acItemBonus`
  — same formula as Phase 44 D-09. All three fields are `number` in `PathbuilderBuild.acTotal`.
- **encounter-persistence.ts**: `loadEncounterIntoCombat` builds `Combatant[]` from `snapshot.combatants`
  (which are `EncounterCombatantRow[]`). After 45-01, `EncounterCombatantRow` has `ac?: number`.
  The spread pattern `...(c.ac != null ? { ac: c.ac } : {})` must be added to each Combatant object.
- **Import change**: Replace `PathbuilderExport` with `PathbuilderBuild` in CharactersPage imports.
  Both are exported from `@engine` (engine/index.ts lines 173-174).

## Threat Model (ASVS L1)

**Phase scope**: Parsing user-imported JSON (Pathbuilder export) and computing AC.

**Threats assessed:**
- **JSON parsing of untrusted input**: `character.rawJson` is stored by the app itself after import validation. The parse is wrapped in `try/catch` — malformed JSON is caught and shows a toast error. Risk: Low.
- **Integer overflow in AC calculation**: `acProfBonus + acAbilityBonus + acItemBonus` are all small integers (PF2e AC rarely exceeds 50). No overflow risk.
- **Type confusion**: Casting unvalidated JSON as `PathbuilderBuild` without runtime validation. Mitigation: error is caught in the try/catch; worst case is a failed "Add to Combat" with a toast error. Acceptable for a DM tool.

**Severity**: No high-severity threats identified for this plan.
</threat_model>
