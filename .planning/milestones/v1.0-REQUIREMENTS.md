# Requirements: Pathfinder 2e DM Assistant

**Defined:** 2026-03-17
**Core Value:** Accurate Pathfinder 2e combat tracking with auto-round processing, condition management, and Foundry VTT PF2e pack compatibility

## v1 Requirements

### Combat Tracker

- [ ] **COMBAT-01**: User can view combat tracker with initiative order
- [ ] **COMBAT-02**: User can add creatures to combat with HP and conditions
- [ ] **COMBAT-03**: User can track HP changes per creature
- [ ] **COMBAT-04**: User can apply/remove conditions (stunned, paralyzed, etc.)
- [x] **COMBAT-05**: System auto-processes rounds with initiative order
- [x] **COMBAT-06**: System auto-decrements duration-based conditions each round
- [x] **COMBAT-07**: System auto-calculates healing/regen and ongoing damage at turn start

### Tauri + SQLite Foundation

- [x] **TAURI-01**: App runs as Tauri 2 desktop application with Vue 3 frontend
- [x] **TAURI-02**: Embedded SQLite database via tauri-plugin-sql
- [x] **TAURI-03**: Drizzle ORM in sqlite-proxy mode for type-safe DB access
- [x] **TAURI-04**: Database migrations run on app startup

### PF2e Data Sync

- [x] **SYNC-01**: User can sync PF2e data from foundryvtt/pf2e GitHub releases
- [x] **SYNC-02**: System downloads release ZIP without requiring git on user machine
- [x] **SYNC-03**: System upserts entities with content-hash change detection
- [x] **SYNC-04**: System removes entities deleted from upstream

### Cross-Reference System

- [x] **XREF-01**: System resolves embedded creature items to canonical entities via slug
- [x] **XREF-02**: Creature detail view shows linked canonical spells/items
- [x] **XREF-03**: NPC-unique abilities render from embedded data

### Description Sanitization

- [x] **DESC-01**: System parses Foundry @UUID links into internal entity links
- [x] **DESC-02**: System renders @Damage, @Check, @Template as readable text

### Sync UI

- [x] **SYNCUI-01**: User sees sync progress with stage indicator and progress bar
- [x] **SYNCUI-02**: User sees current data version after sync
- [x] **SYNCUI-03**: Sync button component with error handling

### Optimizations

- [x] **OPT-01**: Batch SQL imports for 28K+ entities
- [x] **OPT-02**: Full-text search via SQLite FTS5
- [x] **OPT-03**: Generated columns for fast entity filtering (level, rarity)

## v2 Requirements

### Campaign Management

- **CAMPAIGN-01**: User can create multiple campaigns
- **CAMPAIGN-02**: User can switch between campaigns

### Character Support

- **CHAR-01**: Import player characters from Foundry
- **CHAR-02**: Track character-specific conditions and resources

### Encounter Builder

- **ENC-01**: User can create encounters with manual creature entry
- **ENC-02**: User can import creatures from imported bestiary
- **ENC-03**: System calculates XP budget from creatures
- **ENC-04**: User can save encounters to campaigns

### Multi-Window UI

- **UI-01**: Application supports multiple draggable windows
- **UI-02**: User can arrange windows in workspace layout
- **UI-03**: Combat tracker displays as primary window
- **UI-04**: Quick action buttons for attacks and saving throws

### DM Notes

- **NOTES-01**: User can create session notes linked to campaigns
- **NOTES-02**: User can edit notes during sessions

### Integration

- **INTEGR-01**: VTT integration for shared combat
- **INTEGR-02**: API for third-party tools

## Out of Scope

| Feature | Reason |
|---------|--------|
| Cloud sync | Local SQLite persistence only for v1 |
| Player-facing features | DM tool only, not player app |
| Real-time multiplayer | Single-user desktop app |
| Mobile version | Desktop-first with Tauri |
