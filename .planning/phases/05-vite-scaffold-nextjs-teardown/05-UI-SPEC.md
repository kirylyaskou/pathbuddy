## AppShell Layout Contract

Phase 5 renders only the AppShell chrome. This defines the structure every subsequent phase builds within.

```
┌────────────────────────────────────────────────────┐
│  AppSidebar (240px fixed)  │  AppHeader (flex row) │
│  ─────────────────────────  ───────────────────────│
│  Nav items (Link to=)      │  <main> Outlet area   │
│                            │  Placeholder content  │
│  Section headers           │  (each page route)    │
│  CommandPalette trigger    │                       │
└────────────────────────────────────────────────────┘
```

**Root container:** `flex h-screen overflow-hidden bg-background`

**Sidebar:** `bg-sidebar text-sidebar-foreground` — fixed 240px width, full height, `overflow-y-auto`

**Main area:** `flex flex-col flex-1 overflow-hidden` — header + scrollable content below

**Active nav item:** Uses `bg-sidebar-accent` background + `text-sidebar-primary` color (gold). Indicator via `useLocation().pathname` match — not `usePathname()`.

**CommandPalette:** Rendered at root level (inside AppShell, outside page area), triggered by sidebar button or keyboard shortcut. Uses `cmdk` + shadcn Command component. `useNavigate()` replaces `useRouter()`.

**ThemeProvider:** `attribute="class"` + `defaultTheme="dark"` + `disableTransitionOnChange`. Wraps entire app in AppProviders, outside AppRouter.

## Implementation Constraints

Critical constraints for the executor — derived from RESEARCH.md:

1. **Tailwind v4 `@theme inline` is mandatory** — placing tokens only in `:root` without `@theme inline` generates no utility classes (`bg-pf-gold` etc. will silently fail). Port globals.css verbatim from D:/parse_data/app/globals.css.

2. **`@custom-variant dark (&:is(.dark *))` must be present** — this is how Tailwind v4 handles the `.dark` class selector variant. Without it, dark mode utility variants do not apply.

3. **`@layer base` reset is required** — `* { @apply border-border outline-ring/50; }` ensures focus rings are visible on dark backgrounds (Tailwind v4 changed default ring from 3px to 1px). Already present in prototype globals.css.

4. **No `tailwind.config.js` or `tailwind.config.ts`** — Tailwind v4 is CSS-first. Config file must not be created. All customization lives in globals.css.

5. **No `postcss.config.mjs`** — Tailwind v4 Vite uses `@tailwindcss/vite` plugin, not the PostCSS path.

6. **`@` alias points to `src/`, not repo root** — `"@/*": ["src/*"]` in tsconfig.json. The prototype has `"@/*": ["./*"]` (repo root) — this must NOT be copied.

7. **`components.json` aliases must use FSD paths** — `"components": "@/shared/ui"` not `"@/components"`. Edit immediately after `npx shadcn@latest init`.

8. **Remove `"use client"` directives** when porting shadcn components — harmless but noisy in Vite (all components are client-only by default).
