# Plan 10-01: Weak/Elite Tier Toggle + MAP Attack Display

<objective>
Add Weak/Elite tier selection when adding creatures (combat + encounter builder) with HP/level preview via engine `getHpAdjustment`, and display MAP modifier sets in creature stat block strikes via engine `buildAttackModifierSets`.
</objective>

## Tasks

<task id="1">
<title>Add TierToggle inline component and wire into BestiarySearchPanel</title>
<read_first>
- src/widgets/bestiary-search/ui/BestiarySearchPanel.tsx (current add flow)
- src/features/combat-tracker/lib/initiative.ts (createCombatantFromCreature signature)
- src/entities/creature/model/types.ts (WeakEliteTier re-export from engine)
- engine/encounter/weak-elite.ts (getHpAdjustment signature and level adjustment)
</read_first>
<action>
1. In `BestiarySearchPanel.tsx`, add local state:
   ```tsx
   const [selectedTier, setSelectedTier] = useState<WeakEliteTier>('normal')
   ```
   Import `WeakEliteTier` from `@/entities/creature` and `getHpAdjustment` from `@engine`.

2. Add a 3-button inline toggle group below the search results, visible when a creature card's Add button area is shown. Use shadcn ToggleGroup or 3 plain buttons styled:
   - Weak: `text-xs px-2 py-1` — muted when not selected
   - Normal: default selected
   - Elite: gold accent when selected (`bg-primary text-primary-foreground`)

3. Show HP adjustment preview next to the toggle:
   ```tsx
   const hpDelta = getHpAdjustment(selectedTier, creature.level)
   // Display: "HP: 45 → 35" for weak, "HP: 45 → 55" for elite, nothing for normal
   ```

4. Modify `handleAdd` to apply tier:
   ```tsx
   const hpDelta = getHpAdjustment(selectedTier, row.level ?? 0)
   const adjustedHp = Math.max(1, (row.hp ?? 0) + hpDelta)
   const combatant = createCombatantFromCreature(
     creature.id,
     creature.name,
     creature.perception,
     adjustedHp,
     combatants
   )
   // Also set maxHp on combatant to adjustedHp before adding
   combatant.maxHp = adjustedHp
   addCombatant(combatant)
   setSelectedTier('normal') // reset after add
   ```

5. Reset tier to 'normal' when search query changes.
</action>
<acceptance_criteria>
- BestiarySearchPanel.tsx contains `getHpAdjustment` import from `@engine`
- BestiarySearchPanel.tsx contains `selectedTier` state with type `WeakEliteTier`
- Selecting Elite on a level 5 creature shows positive HP delta text
- Selecting Weak on a level 5 creature shows negative HP delta text
- After adding a creature, selectedTier resets to 'normal'
- Added combatant's hp and maxHp reflect the tier adjustment
</acceptance_criteria>
</task>

<task id="3">
<title>Add MAP attack modifier display to CreatureStatBlock strikes</title>
<read_first>
- src/entities/creature/ui/CreatureStatBlock.tsx (strikes rendering, lines 105-142)
- src/entities/creature/model/types.ts (CreatureStatBlockData.strikes type)
- engine/statistics/creature-statistics.ts (buildAttackModifierSets — need to understand its signature)
- engine/index.ts (exports buildAttackModifierSets)
</read_first>
<action>
1. In `CreatureStatBlock.tsx`, for each strike, compute MAP values inline. The strike already has `modifier` (the base attack bonus). MAP follows standard PF2e rules:
   - MAP 0: +modifier (already shown)
   - MAP -5: modifier - 5
   - MAP -10: modifier - 10

   Check if strike has the `agile` trait — if so, MAP penalties are -4/-8 instead of -5/-10.

2. Add a row below each strike's damage line showing MAP modifiers:
   ```tsx
   <div className="mt-1 flex gap-3 text-xs text-muted-foreground">
     <span className="font-mono">
       MAP: <span className="text-primary">+{strike.modifier}</span>
       {' / '}
       <span className="text-muted-foreground">{formatModifier(map1)}</span>
       {' / '}
       <span className="text-muted-foreground">{formatModifier(map2)}</span>
     </span>
   </div>
   ```

   Where `formatModifier(n)` returns `+N` for positive, `N` for negative.

3. Helper logic inside the component:
   ```tsx
   const isAgile = strike.traits.includes('agile')
   const map1 = strike.modifier - (isAgile ? 4 : 5)
   const map2 = strike.modifier - (isAgile ? 8 : 10)
   ```

Note: Using inline MAP calculation from strike modifier + agile trait rather than engine `buildAttackModifierSets` because the stat block only has the pre-computed modifier number, not a full Creature object needed by the engine's CreatureStatistics adapter. The MAP rules are simple arithmetic and this avoids an unnecessary architecture change. INT-05 requirement is satisfied — MAP is displayed.
</action>
<acceptance_criteria>
- CreatureStatBlock.tsx renders MAP row for each strike
- A strike with modifier +15 and no agile trait shows: MAP: +15 / +10 / +5
- A strike with modifier +12 and agile trait shows: MAP: +12 / +8 / +4
- MAP row uses `text-xs text-muted-foreground` styling
- MAP row is below the Damage line inside each strike block
</acceptance_criteria>
</task>
