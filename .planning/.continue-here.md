## Bug Summary

**Error:** `Maximum update depth exceeded` — `setRef → dispatchSetState` in Radix compose-refs (`chunk-ALRU7VAZ.js`)

**Root cause:** `ConditionCombobox` (committed HEAD before fix) used `<Dialog modal={false}> + <Tabs> + <TabsContent>`.
- `TabsContent` uses Radix `Presence` internally
- Dialog itself also wraps content in `Presence` (via DialogPortal)
- Nested Presence levels → `composeRefs` callback created new instance on every Zustand sync re-render
- Sequence: `applyCondition()` → Zustand `useSyncExternalStore` sync re-render → unstable `composeRefs` → `setNode(null/node)` every render → infinite loop

**Why `modal={false}` alone was insufficient:** It only prevents `react-remove-scroll` from activating (a separate loop type). The Presence/composeRefs loop does NOT depend on modal mode.

## Key Design Decision

**createPortal + flushSync** is the correct pattern for any component that:
1. Lives inside a tree with Zustand subscriptions, AND
2. Calls Zustand store mutations, AND
3. Uses Radix components with Presence

`flushSync(close)` ensures the portal is fully unmounted **before** the Zustand mutation fires → no Presence in tree when `useSyncExternalStore` triggers sync re-render.
