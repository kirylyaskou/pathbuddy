## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, rank badge padding |
| sm | 8px | Toggle button padding, compact row gaps |
| md | 16px | Card section padding, select gap |
| lg | 24px | PCCombatCard section spacing |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: initiative input width is fixed `~3rem (48px)` — not on scale; tier-chip toggle `px-2 py-0.5` (8px/2px) matches existing BestiarySearchPanel chip pattern.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 | 1.5 |
| Label | 12px (text-xs) | 400 | 1.4 |
| Heading | 16px (text-base) | 600 | 1.4 |
| Display | 20px (text-xl) | 600 | 1.3 |

Two weights only: **400** (regular — body, labels, muted text) and **600** (semibold — names, headings, active emphasis).

**PCCombatCard specifics:**
- Character name: `text-base font-semibold` (weight 600)
- Class + level line: `text-xs text-muted-foreground` (weight 400)
- Stat values (ability mods, saves): `text-sm font-mono` via JetBrains Mono — matches existing CreatureStatBlock pattern (weight 400)
- Stat labels (STR / DEX / etc.): `text-xs text-muted-foreground uppercase tracking-wide` (weight 400)
- Skill rank badges (U/T/E/M/L): `text-[10px] font-semibold` (weight 600, sub-label implementation detail)

**InitiativeRow inline edit:**
- Initiative number display: existing `text-sm font-mono` (weight 400)
- Inline input: `text-sm font-mono text-center` — visually identical to display to avoid layout shift (weight 400)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(0.96 0.015 70) light / oklch(0.14 0.025 50) dark | Page background, panel backgrounds |
| Secondary (30%) | `--card` oklch(0.98 0.012 65) light / oklch(0.17 0.03 50) dark | PCCombatCard surface, search panel background |
| Accent (10%) | `--primary` oklch(0.55 0.14 65) light / oklch(0.72 0.18 75) dark | Active toggle highlight bg, skill rank badges |
| Destructive | `--destructive` oklch(0.50 0.20 25) | Destructive actions only — not used in this phase |

Accent reserved for: active toggle button background (`bg-primary text-primary-foreground`), skill rank badge E/M/L (expert+ highlight uses `text-pf-skill-expert`).

**Hazard visual identity (established in Phase 28–30):**
- AlertTriangle icon: `text-amber-500` — matches existing hazard rows in EncounterCreatureList
- Hazard dice roll button: `text-amber-500 hover:text-amber-400`
- Hazard initiative toast: neutral — no amber override needed on toast itself

**Inline initiative edit:**
- Input border on focus: `ring-1 ring-ring` (uses `--ring` token = `--primary`)
- No background change — input blends with row background

**Left panel toggle [Bestiary | Hazards | Characters]:**
- Inactive: `bg-transparent text-muted-foreground hover:bg-muted hover:text-foreground`
- Active: `bg-primary text-primary-foreground`
- Copy the exact `px-2 py-0.5 text-xs rounded transition-colors` from BestiarySearchPanel tier chips

**Skill rank badge colors:**
- U (untrained): `text-muted-foreground` — dim
- T (trained): `text-foreground` — neutral
- E (expert): `text-pf-skill-expert` (oklch(0.50 0.16 235) — blue)
- M (master): `text-pf-gold` — gold
- L (legendary): `text-primary font-bold` — primary gold

## Component Specs

### 1. Left Panel Toggle [Bestiary | Hazards | Characters]

```
[Bestiary] [Hazards] [Characters]
```

- Container: `flex gap-1 p-1` at top of left panel, above search input
- Each button: `px-2 py-0.5 text-xs rounded transition-colors`
- Active: `bg-primary text-primary-foreground`
- Inactive: `bg-transparent text-muted-foreground hover:bg-muted hover:text-foreground`
- No border on buttons — matches tier chip pattern
- Tab state is local to left panel (not in global store)

### 2. Characters Tab Row

```
⊕  Alira Stormborn  —  Magus 7
```

- Icon: `UserPlus` or `Plus` size-3, `text-muted-foreground`
- Name: `text-sm font-medium`
- Class + level: `text-xs text-muted-foreground` inline after em dash
- Row: `flex items-center gap-2 px-2 py-1 rounded hover:bg-muted cursor-pointer`
- Click adds to combat (with encounter picker if >1 tabs open)

### 3. Encounter Picker Dropdown (Characters tab + /characters page)

- shadcn `DropdownMenu` or `Select` component
- Trigger: "Add to Combat" button (existing style)
- Items: encounter tab names
- Only shown when `openTabs.length > 1`

### 4. PCCombatCard

Layout — vertical sections, no tabs:

```
┌─────────────────────────────┐
│ Alira Stormborn              │  ← name text-base font-semibold
│ Magus 7                      │  ← class + level text-xs muted
├─────────────────────────────┤  ← border-b border-border/50
│ HP  45 / 60    AC  18        │  ← two stats inline
├─────────────────────────────┤
│ STR  DEX  CON  INT  WIS  CHA │
│ +2   +4   +1   +5   +0   +1  │  ← 6-col grid, font-mono
├─────────────────────────────┤
│ Saves                        │
│ Fort +8   Ref +11   Will +6  │  ← 3-col inline, font-mono
├─────────────────────────────┤
│ Skills                       │
│ [T] Acrobatics    +9         │
│ [E] Arcana       +12         │  ← rank badge + name + modifier
│ ...                          │    ScrollArea if >8 skills
└─────────────────────────────┘
```

- Outer: `rounded-lg border border-border bg-card p-3 space-y-3`
- Section dividers: `border-t border-border/50 pt-3`
- HP/AC row: `flex gap-4` — each stat `flex flex-col items-center`
- Ability score grid: `grid grid-cols-6 gap-1 text-center`
- Saves: `flex gap-4 text-sm`
- Skills list: `space-y-1` inside `ScrollArea max-h-48`
- Skill row: `flex items-center gap-2 text-sm`
- Rank badge: `inline-flex items-center justify-center w-4 h-4 rounded text-[10px] font-semibold bg-muted`

### 5. Inline Initiative Edit (InitiativeRow)

- Display: `<span className="cursor-pointer hover:underline font-mono text-sm" onClick={startEdit}>`
- Edit: `<input type="number" className="w-12 text-center text-sm font-mono bg-transparent border rounded px-1 ring-1 ring-ring outline-none" />`
- On Enter / blur: save → `setInitiative` + `reorderInitiative`
- On Escape: cancel — restore display
- No layout shift: input width `w-12` matches typical 2-digit number display width

### 6. Hazard in InitiativeRow

- Icon swap: `AlertTriangle` size-4 `text-amber-500` (replaces User/Skull)
- Dice button: appears between action buttons and initiative number
  - `Button variant="ghost" size="icon" className="h-5 w-5 text-amber-500 hover:text-amber-400"`
  - Icon: `Dices` size-3 from lucide-react
  - Only visible when `combatant.isHazard === true`

### 7. Creature Type Select (BestiarySearchPanel + EncounterCreatureSearchPanel)

- shadcn `Select` component
- Trigger: `text-xs h-7` — compact, matches search input height
- Position: after search input, same row or directly below depending on available width
- Default value: `""` (renders as "All types")
- Options: 20 entries from `CREATURE_TYPES` array
- No "clear" button needed — "All types" option serves as reset
