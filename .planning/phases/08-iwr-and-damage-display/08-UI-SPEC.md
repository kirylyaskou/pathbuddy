## IWR Section Layout Contract

Position: after saves row (Fort/Ref/Will), before languages row. Matches PF2e CRB stat block order.

**Visibility rule:** Entire IWR section hidden (`v-if`) when creature has no immunities, no weaknesses, and no resistances. No empty row, no "none" placeholder.

**Three sub-rows rendered independently (each hidden when array is empty):**

```
Immunities  Fire, Cold Iron, Critical Hits
Weaknesses  Fire 5, Cold Iron 3
Resistances  Physical 5 (except adamantine), Fire 3
```

Row structure per sub-row:
- Label: `text-xs text-stone-400` — "Immunities", "Weaknesses", "Resistances"
- Values: comma-separated inline spans
  - Immunity types: `text-sm text-stone-300` — title-cased slug, no numeric value
  - Weakness entries: type label `text-sm text-stone-300` + value `text-sm font-bold text-crimson-light`
  - Resistance entries: type label `text-sm text-stone-300` + value `text-sm font-bold text-emerald-400`
  - Exception parenthetical: `text-xs text-stone-400` appended after value (e.g. "(except adamantine)")

**Type label formatting:** slug split on `-`, each word title-cased. Examples:
- `fire` → "Fire"
- `cold-iron` → "Cold Iron"
- `critical-hits` → "Critical Hits"
- `precision` → "Precision"

**Unknown type fallback:** If engine taxonomy does not recognize the slug, display the title-cased slug as-is with no icon. No error state.

**Spacing:** `mt-1` between IWR section and saves row. `space-x-1` between label and value list. Individual type entries separated by `, ` (comma + space) as plain text, not chips/tags.

## Damage Category Icon Contract

Position: after strike item name, before the canonical cross-reference link button.

**Three icons (inline SVG, 16x16px, `w-4 h-4`, `text-stone-400` stroke color):**

| Category | Icon | SVG Description |
|----------|------|-----------------|
| physical | sword | Single diagonal sword: line from bottom-left to top-right with crossguard perpendicular line and pommel dot — `stroke-linecap="round" stroke-linejoin="round" stroke-width="2"` |
| energy | flame | Teardrop flame shape with inner curl: outer path traces a flame silhouette, inner small curve at base — same stroke style |
| other | sparkle | Four-pointed star / sparkle: two crossed lines, each with a small perpendicular accent midpoint — same stroke style |

Icon color: `text-stone-400` (informational, not semantic). Do NOT use crimson or emerald for these icons.

Icon sizing: `w-4 h-4` (16px). This matches the canonical cross-reference link SVG already in StatBlock.vue at line 396.

**Damage type label:** `text-xs text-stone-400` rendered immediately after icon with `gap-1` between icon and text. Example: `[sword icon] Slashing`.

**Layout:** `flex items-center gap-1` wrapping `[icon] [type label]` — inline within the item name row `flex items-center gap-2` already established in StatBlock.vue line 386.

**Parsing path:** `item.embedded.system.damage.damageType` from Foundry raw data → `DamageCategorization.getCategory(damageType)` → one of `'physical' | 'energy' | 'other'`.

**Fallback:** If `damageType` is missing or `getCategory()` returns undefined, render nothing (no icon, no label). Do NOT render a broken icon placeholder.

**Strike type scope:** Apply to melee and ranged strike sections only (SECTION_CONFIG keys `'melee'` and `'ranged'`). Do not apply to spellcasting, actions, or equipment.
