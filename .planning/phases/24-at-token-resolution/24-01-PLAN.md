# Plan 24-01: @-Token Resolution Improvements

**Phase:** 24 — @-Token Audit + Fix
**Requirements:** ATRES-01, ATRES-02, ATRES-03
**Status:** Complete

## Goal

Audit all @-token types appearing in item and creature descriptions; ensure they resolve to human-readable text. Key gap: @UUID[Compendium.pf2e.equipment.Item.XXX] without alias currently renders the raw Foundry ID as the label — resolve to item name from DB.

## Changes

- `src/entities/creature/model/mappers.ts` — extend `resolveFoundryTokens(text, lookupFn?)`:
  - New optional async overload `resolveFoundryTokensAsync(text, db)` for use at import time
  - `@UUID[Compendium.pf2e.equipment.Item.{id}]` without alias → lookup name from `items` table by foundry id; fallback to last path segment
  - `@UUID[Compendium.pf2e.spells-srd.Item.{id}]` without alias → lookup name from `spells` table
  - `@UUID[Compendium.pf2e.conditions.Item.{id}]` without alias → lookup name from `entities` table (type='condition')
  - `@UUID[Compendium.pf2e.actionspf2e.Item.{id}]` without alias → lookup from `entities` table

- `src/shared/api/sync.ts`:
  - After extractAndInsertItems(), call a post-processing pass over creature descriptions and item descriptions to resolve unaliased @UUID tokens using DB lookups
  - New `resolveUUIDTokensInDescriptions()` function: batch-updates raw description strings in `items.description`, `spells.description`, and stored entity descriptions

- `src/entities/creature/ui/CreatureStatBlock.tsx` + `src/entities/item/ui/ItemCard.tsx`:
  - `resolveFoundryTokensForDisplay(text)` — synchronous, uses already-resolved descriptions; handles remaining inline tokens: `@Localize`, `[[/act]]`, `[[/br]]`, `{Nfeet}` patterns
  - Audit: enumerate all unresolved token types appearing in descriptions by running sample queries and checking for remaining @ or [[ patterns

## Notes

- Import-time resolution preferred over runtime — descriptions in DB should already be clean
- Runtime resolver kept as fallback for any tokens missed at import time
- Items from `equipment-effects` pack: these are `effect` type items — store in `items` table with item_type='effect'; descriptions may reference conditions via @UUID
- Audit query: `SELECT description FROM items WHERE description LIKE '%@UUID%' AND description NOT LIKE '%]{%' LIMIT 20` — finds unaliased UUID refs
- This phase is a polish/correctness pass — all prior functionality works without it; items just show raw IDs in rare cases
