## Objective

Add the `ac` field to the data layer: create the SQLite migration, extend the `Combatant`
runtime type and `EncounterCombatantRow` persistence type, and update `saveEncounterCombatants` /
`loadEncounterCombatants` to persist and restore it. This plan is the prerequisite for all
Wave 2 plans.

## Context

- `encounter_combatants` table currently has 14 columns; `ac` will be added as a nullable INTEGER.
- `ALTER TABLE … ADD COLUMN` is non-destructive in SQLite — existing NPC rows get `NULL`, which maps to `undefined` in TypeScript (correct: NPC AC lives in the stat block panel, not this field).
- Migration system uses `import.meta.glob` in `src/shared/db/migrate.ts` — any new `.sql` file in `migrations/` is automatically picked up on next app start. No registration step needed.
- `saveEncounterCombatants` currently INSERTs 14 positional params. The INSERT and params array must both add `ac` as the 15th entry.
- `loadEncounterCombatants` uses `SELECT *` — the `ac` column will appear in results automatically. The in-memory type annotation and the return mapping both need updating.
- `EncounterCombatantRow.ac` should be `?: number` (optional) — NPC rows will have `null` from DB, mapped to `undefined`.
- `Combatant.ac` should be `?: number` for the same reason.
