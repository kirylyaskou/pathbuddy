## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), pill chip inner padding (`px-1`) |
| sm | 8px | Compact element spacing, chip row gap (`gap-2`) |
| md | 16px | Default panel padding (`px-4`), section padding |
| lg | 24px | Section vertical padding (`py-6`) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Filter bar row height: 40px (10 Tailwind units, `h-10`) — matches existing sidebar nav row height from AppSidebar.vue
- Browser result row height: 40px fixed (`item-size="40"`) — required by virtua VList for virtualisation
- Active filter chip dismiss button: 24px (`w-6 h-6`) — minimum touch/click target for × affordance
- Toggle expand button area: 16px icon inside `px-4 py-2` row — 40px effective row height covers click target

## Typography

| Role | Size | Weight | Line Height | Class | Source |
|------|------|--------|-------------|-------|--------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm` | CreatureCard.vue stat text |
| Label / badge | 12px | 400 (normal) | 1.25 | `text-xs` | CreatureCard.vue condition badges, ConditionBadge.vue |
| Panel heading | 20px | 700 (bold) | 1.2 | `text-xl font-display font-bold` | CreatureDetailPanel.vue h2 |
| Section / component heading | 16px | 700 (bold) | 1.3 | `text-base font-display font-bold` | CreatureCard.vue h3 |

Rules:
- Headings always use `font-display` (Cinzel). Body and labels use system-ui (no class needed — Tailwind default).
- Maximum 2 weights across the entire phase: 400 (body/label) and 700 (bold — headings and WeakEliteSelector selected segment). No third weight is used.
- Font sizes for this phase: 12px (xs labels, badges, HP delta), 14px (list rows, filter inputs, chip text), 16px (filter bar section label, component subheadings), 20px (slide-over panel title — reused, not new).

## Color

| Role | Value | Tailwind token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#1a1a1a` | `bg-charcoal-800` | Main content area backgrounds, browser list rows |
| Secondary (30%) | `#2a2a2a` | `bg-charcoal-600` | Filter panel background, panel surfaces, slide-over |
| Accent (10%) | `#c9a84c` | `bg-gold` / `text-gold` / `border-gold` | Selected filter chip × dismiss hover, WeakEliteSelector selected segment fill, active filter chip border accent, sidebar active-link indicator (pre-existing) |
| Destructive | `#8b2020` | `bg-crimson` | Not used in Phase 03 (no destructive actions) |

Accent reserved ONLY for:
1. WeakEliteSelector selected segment background (`bg-gold text-charcoal-950`)
2. Active filter chip dismiss button hover (`hover:text-gold`)
3. Filter toggle button gold border when filters are active (`border-gold`)
4. Empty state "Run sync" link text (`text-gold`)

Secondary semantic colors (informational, not accent):
- `bg-charcoal-700` — hover state for list rows and collapsed filter toggle (`hover:bg-charcoal-700`)
- `bg-charcoal-900` — deepest surface for stats row in detail panel, section headers
- `text-stone-300` — secondary text (family tag, rarity text in list rows)
- `text-stone-400` — muted/placeholder text (HP delta label, level badge text)
- `text-stone-500` — disabled/faint (version footer, empty state body)
- `border-charcoal-700` — row separators in virtualised list
- `border-charcoal-500` — panel borders, filter panel borders

Rarity dot color map (semantic indicators, not accent):
- Common: `text-stone-400` (no special color — neutral)
- Uncommon: `text-amber-500`
- Rare: `text-blue-400`
- Unique: `text-purple-400`

## Visual Hierarchy

Primary visual anchor: the CreatureBrowser list. It occupies the majority of vertical space and is where user attention lands after initial load — the virtualised rows are the dominant element on screen. The EntityFilterBar toggle is the secondary focal point: it sits above the list and becomes the user's first interaction point when narrowing results. All other controls (WeakEliteSelector, chip summary) are tertiary and appear only in contextual sub-panels or downstream in the composition hierarchy.

## Component Interaction Contracts

> This section supplements the standard template fields for this phase's three new components.

### EntityFilterBar

**States:**

| State | Visual |
|-------|--------|
| Collapsed, no active filters | Single toggle row: `bg-charcoal-600 border-b border-charcoal-500`, text "Filters" + chevron-down icon (16×16 SVG) |
| Collapsed, with active filters | Toggle row + chip summary row below. Toggle button shows `border-gold` outline to signal active state. Chip row: `flex flex-wrap gap-1 px-2 py-1` |
| Expanded | Toggle row + filter controls panel (max-h-96, transition `duration-200 ease-in-out`) |
| Active filter chip | `bg-charcoal-700 text-stone-300 border border-charcoal-500 rounded-full px-2 py-0.5 text-xs` |
| Active filter chip × hover | `hover:text-gold` on × button |

**Filter controls layout (expanded):**

| Control | Type | Width | Placement |
|---------|------|-------|-----------|
| Entity type | Native `<select>` | 50% | Row 1, left |
| Rarity | Native `<select>` | 50% | Row 1, right |
| Level min | `<input type="number">` | 30% | Row 2, left |
| Level max | `<input type="number">` | 30% | Row 2, center |
| Auto button | `<button>` | 40% | Row 2, right |
| Auto config (inline, shown when Auto active) | Party size + party level `<input type="number">` | Full row | Row 3, conditional |
| Family | Native `<select>` | 100% | Row 4, only when entityType = 'creature' |
| Tags/traits search | `<input type="text">` + datalist dropdown | 100% | Row 5 |
| Tags selected pills | Chip row (same style as active filter chips) | 100% | Row 6, only when tags selected |
| AND / OR toggle | `<button>` toggle | auto | Inline right of tag input row label |

Input styling: `bg-charcoal-700 border border-charcoal-600 rounded px-2 py-1 text-sm text-stone-200 focus:border-gold focus:outline-none` on all form inputs and selects.

**Collapsible panel transition:**
```
:class="['overflow-hidden transition-all duration-200 ease-in-out', expanded ? 'max-h-96 opacity-100' : 'max-h-0 opacity-0']"
```
Source: RESEARCH.md code example — Tailwind v3 compatible max-h trick.

**AND/OR toggle:**
- Default: AND (label: "Match all traits")
- Toggled: OR (label: "Match any trait")
- Visual: small `<button>` with `bg-charcoal-600 text-xs text-stone-400 px-2 py-0.5 rounded border border-charcoal-500` default; `bg-charcoal-700 text-stone-200 border-gold` when toggled

### CreatureBrowser

**States:**

| State | Visual |
|-------|--------|
| Loading | Skeleton pulse rows: `animate-pulse bg-charcoal-700 h-10 rounded mx-2 my-1` — 5 skeleton rows |
| Results (normal) | virtua VList, fixed row height 40px, rows per RESEARCH.md code example |
| Empty (no match) | See copywriting contract below |
| Result row default | `bg-charcoal-800 border-b border-charcoal-700 hover:bg-charcoal-700 cursor-pointer transition-colors` |
| Result row hovered | `bg-charcoal-700` |

**Result row layout (h-10 = 40px fixed):**
```
[Name — flex-1 text-sm text-stone-100 truncate] [Lvl badge] [Rarity dot] [Family tag — text-xs text-stone-500]
```
- Name: `text-sm text-stone-100 truncate flex-1`
- Level badge: `text-xs px-1.5 py-0.5 rounded bg-charcoal-600 text-stone-300` — content: `Lvl {n}` (or `Lvl –` if null)
- Rarity dot: 6px circle (`w-1.5 h-1.5 rounded-full`) using rarity color map above
- Family tag: `text-xs text-stone-500` — omitted if null

**Result count display:**
- Shown above the list, below the filter bar: `text-xs text-stone-400 px-3 py-1` — copy: `{n} results` (or `Showing 100 of {total}` when capped)

### WeakEliteSelector

**Layout:** Horizontal flex, three buttons + HP delta label to the right.

**Button states:**

| State | Class |
|-------|-------|
| Unselected | `bg-charcoal-700 text-stone-300 hover:bg-charcoal-600 px-3 py-1 text-sm rounded transition-colors` |
| Selected | `bg-gold text-charcoal-900 font-bold px-3 py-1 text-sm rounded transition-colors` |

**HP delta label:** `ml-2 text-xs text-stone-400` — content: `Normal` when tier=normal; `Elite: +{n} HP` or `Weak: {n} HP` (n is negative, e.g. `Weak: -20 HP`) for other tiers.

**Button labels:** `Weak` | `Normal` | `Elite` (sentence case, not uppercase)

**Order:** [Weak] [Normal] [Elite] — left to right, default selection is Normal.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Filter toggle (collapsed, no filters) | `Filters` | With chevron-down icon |
| Filter toggle (collapsed, filters active) | `Filters (active)` | Gold border on toggle button |
| Auto level range button | `Auto` | Reveals inline party config on click |
| Auto party config label | `Party level` and `Size` | Inline number inputs |
| AND/OR toggle (AND) | `Match all traits` | Default |
| AND/OR toggle (OR) | `Match any trait` | Toggled state |
| Result count | `{n} results` | Below filter bar |
| Result count capped | `Showing 100 of {total}` | When default 100-row cap applies |
| Empty state heading | `No matches` | `text-sm font-bold text-stone-300` |
| Empty state body (filters active) | `No entities match the current filters. Clear a filter to broaden your search.` | `text-xs text-stone-400` |
| Empty state body (no sync data) | `No data synced yet. Go to Sync Data to import PF2e content.` | `text-xs text-stone-400`; "Sync Data" is a RouterLink in `text-gold` |
| Loading state | (skeleton rows only — no text label needed) | Pulse animation per component spec above |
| Error state (query failed) | `Could not load results. Try again or restart the app.` | `text-sm text-crimson-light px-4 py-4` |
| Slide-over open (row click) | (no label — direct open action) | Reuses existing CreatureDetailPanel |
| WeakEliteSelector HP label | `Normal` / `Elite: +{n} HP` / `Weak: -{n} HP` | Computed from getHpAdjustment() |
| Family filter label | `Family` | Shown only when entityType = 'creature' |
| Tags filter label | `Traits` | NOT "Tags" — PF2e uses "traits" terminology |
| Tags filter placeholder | `Search traits…` | Input placeholder |
| Tag chip dismiss aria | `Remove {trait} trait` | aria-label on × button |
| Level range label | `Level` | Followed by min/max inputs |

Destructive actions: none in this phase. No confirmation copy required.
