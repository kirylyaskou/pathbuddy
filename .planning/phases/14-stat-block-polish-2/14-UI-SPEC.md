## Color

### Base 60/30/10 Split

| Role | Dark Mode | Light Mode | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `oklch(0.14 0.025 50)` | `oklch(0.96 0.015 70)` | Page background, card interior |
| Secondary (30%) | `oklch(0.17 0.03 50)` | `oklch(0.98 0.012 65)` | Cards, secondary surface, stat block bg |
| Accent (10%) | `oklch(0.72 0.18 75)` | `oklch(0.55 0.14 65)` | `--primary` / `pf-gold` — reserved for accent elements below |

Accent (`text-primary` / `bg-primary`) reserved for:
- Strike attack modifier values (`+N`)
- MAP first-attack value
- Section header left border (`border-primary/50`)
- Section header gradient background (`from-primary/10`)
- Skill modifier values

### Visual Hierarchy & Focal Point

**Primary focal point:** The HP value (`text-pf-threat-extreme` red) in the Core Stats row is the highest-contrast element — the DM's eye lands here first to assess creature health threat.

**Secondary focal points (in order):** AC (gold) → Saves (green, grouped) → Perception (dim gold) → Section headers (gradient strip) → Strike modifier (gold, `text-primary`).

Ability descriptions and skill values recede visually (muted) to avoid competing with the critical combat stats.

## Copywriting Contract

Phase 14 is display-only (no user actions on the stat block). Copywriting contract covers inline text labels and empty/error states.

| Element | Copy |
|---------|------|
| Strikes section header | "Strikes" |
| Abilities section header | "Abilities" |
| Skills section header | "Skills" |
| Empty strikes state | "No strikes listed" |
| Empty abilities state | "No abilities listed" |
| Damage display (multi-type) | `{dice} {type} plus {dice} {type}` (existing `formatDamage` pattern) |
| Weapon group display | "Group: {group}" — shown as small badge below strike name |
| MAP label | "MAP:" prefix, then values separated by " / " (existing) |
| Calculated skill indicator | Rendered at 40% opacity (`opacity-40`) — no label change |
| Token fallback (unresolved) | Silently drop unknown tokens — never show raw `[[...]]` or `{...}` syntax to user |

### Token Rendering Rules (resolveFoundryTokens additions)

| Input Pattern | Output | Example |
|--------------|--------|---------|
| `[[/act slug]]` | Capitalize slug, hyphens to spaces | `[[/act shove]]` → "Shove" |
| `[[/act slug #id]]` | Capitalize slug | `[[/act stride #stride]]` → "Stride" |
| `[[/br expr #label]]{display}` | Use `{display}` text only | `[[/br 2d6+5 #hit]]{2d6+5}` → "2d6+5" |
| `[[/br expr]]` (no display) | Use expr as-is | `[[/br 1d6]]` → "1d6" |
| `{Nfeet}` where N is digits | "N feet" | `{30feet}` → "30 feet" |
| `{Nfoot}` variant | "N feet" | `{5foot}` → "5 feet" |

## Phase 14 Implementation Scope Summary

This spec covers exactly these changes:

**1. Token rendering (resolveFoundryTokens in `src/entities/creature/model/mappers.ts`)**
- Add `[[/act ...]]` handler (with and without `{display}` alias)
- Add `[[/br ...]]{display}` handler
- Add `{Nfeet}` / `{Nfoot}` handler

**2. Fighter's Fork weapon data (`formatDamage` + strike mapper)**
- Read `item.system.group` and expose as `group?: string` on strike type
- `additionalDamage` array from `item.system.additionalDamage` appended in `formatDamage`
- Ensure traits array includes all entries from `item.system.traits.value`

**3. Color system (`CreatureStatBlock.tsx`)**
- `StatItem` for AC: add `text-pf-gold` class
- `StatItem` for Fort/Ref/Will: add `text-pf-threat-low` class
- `StatItem` for Perception: add `text-pf-gold-dim` class
- Section header collapsible triggers: add gradient + left-border classes
- Ability traits: change from `bg-secondary/80 text-secondary-foreground` to `bg-primary/10 text-primary border border-primary/20`
- Damage type text in `formatDamage` display: wrap type in `<span className="text-pf-blood">` — requires ability description to render HTML or split damage string
- Weapon group badge: render below strike name if `group` is present

**Note on damage type coloring**: Damage type labels are embedded in a formatted string (`"2d6 fire plus 1d4 bleed"`). To color them, either:
- Option A: Return structured data from `formatDamage` and render in TSX (preferred — clean)
- Option B: Apply color to all text after dice pattern (regex post-processing in component)
- **Decision: Option A** — `formatDamage` returns `{ formula: string; type: string }[]`; component renders each part with type colored `text-pf-blood`
