# Plan 48-01: Description Sanitization

## Objective

Eliminate all unresolved Foundry @-tokens from item and spell description surfaces.
Two-part fix: (1) add `@item.*` stripping to the canonical token resolver, (2) switch
two drawer components from bare `stripHtml` to `sanitizeFoundryText` so all existing
token handlers run at render time.

## must_haves

- [ ] `@item.rank` and `@item.level` no longer appear literally in any displayed text
- [ ] `ItemReferenceDrawer` uses `sanitizeFoundryText` instead of `stripHtml`
- [ ] `SpellReferenceDrawer` uses `sanitizeFoundryText` instead of `stripHtml`
- [ ] `@UUID`, `@Check`, `@Template`, `@Damage` tokens resolve correctly in both drawers
- [ ] No import from `@/shared/lib/html` remains in either drawer

## Tasks

<task id="T01" title="Add @item.* stripping to resolveFoundryTokens">

<read_first>
- src/shared/lib/foundry-tokens.ts
</read_first>

<action>
In `src/shared/lib/foundry-tokens.ts`, add the following line immediately BEFORE the
existing catch-all line `text = text.replace(/@\w+\[[^\]]*\](?:\{[^}]*\})?/g, '')`:

```ts
  // @item.rank, @item.level, and other @item.* inline references — strip entirely
  text = text.replace(/@item\.\w+/g, '')
```

The final two lines of `resolveFoundryTokens` should become:

```ts
  // @item.rank, @item.level, and other @item.* inline references — strip entirely
  text = text.replace(/@item\.\w+/g, '')
  // Strip remaining unresolved @ tokens
  text = text.replace(/@\w+\[[^\]]*\](?:\{[^}]*\})?/g, '')
  return text
```

No other changes to this file.
</action>

- The `@item\.\w+` replacement appears BEFORE the catch-all `@\w+\[` replacement
- `resolveFoundryTokens` still returns `text` as its final statement
</acceptance_criteria>

</task>
