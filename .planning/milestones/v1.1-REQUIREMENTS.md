# Requirements: Pathfinder 2e DM Assistant

**Defined:** 2026-03-20
**Core Value:** Accurate Pathfinder 2e combat tracking with auto-round processing, condition management, and Foundry VTT PF2e pack compatibility.

## v1.1 Requirements

Requirements for Milestone v1.1: Compendium & Combat Workspace.
Continuing from v1.1 Phase 01 (Bug Fix and UI Improvements — complete).

### Compendium

- [x] **COMP-01**: User can browse all synced entities on a dedicated Compendium page grouped by entity type (creatures, spells, hazards, items, feats, etc.)
- [x] **COMP-02**: User can filter entities by type on the Compendium page
- [x] **COMP-03**: User can filter entities by level range (1–20) on the Compendium page
- [x] **COMP-04**: User can filter entities by rarity (common/uncommon/rare/unique) on the Compendium page
- [x] **COMP-05**: User can search entities by name using full-text search on the Compendium page
- [x] **COMP-06**: User can filter creatures by family type (e.g. Dragon, Undead, Goblin) — requires JSON path verification (`$.system.details.family`) before implementing
- [x] **COMP-07**: User can filter entities by tags/traits — requires `json_each()` query pattern on raw_data traits array
- [x] **COMP-08**: User can open full entity stat block in slide-over panel from Compendium

### Combat Workspace

- [x] **WORK-01**: Combat tracker page uses a 3-panel layout — creature browser (left ~1/3) | combat tracker (center ~1/3) | creature detail (right ~1/3)
- [x] **WORK-02**: User can search and filter creatures in the creature browser panel (type/level/rarity/name/family/tags filters)
- [x] **WORK-03**: User can add one or more creatures to combat from the browser (quantity selector, auto-numbered if same name: "Goblin 1", "Goblin 2")
- [x] **WORK-04**: User can apply weak / normal / elite adjustment when adding a creature (HP auto-adjusted per official PF2e 12-bracket tier table from Archives of Nethys)
- [x] **WORK-05**: Added weak/elite creatures are labeled in the combat tracker (e.g. "Elite: Adult Black Dragon")
- [x] **WORK-06**: Clicking a combat tracker row populates the right detail panel with the creature's full stat block plus live combat state (current HP, conditions, initiative)
- [x] **WORK-07**: Manual "Add Creature" form removed — creature browser is the sole entry point for adding to combat (last, after full validation of new flow)
