# Requirements: Pathfinder 2e DM Assistant

**Defined:** 2026-03-24
**Core Value:** Accurate Pathfinder 2e combat tracking with auto-round processing, condition management, and Foundry VTT PF2e pack compatibility.

## v2.0 Requirements

Requirements for v2.0 PF2e Game Logic Engine. Each maps to roadmap phases.

### XP & Encounter Budget

- [x] **XP-01**: XP lookup tables for creature, variant (pwol), simple hazard, complex hazard level differences
- [x] **XP-02**: calculateCreatureXP maps party-vs-creature level difference to XP value
- [x] **XP-03**: getHazardXp handles simple and complex hazard XP with level difference
- [x] **XP-04**: generateEncounterBudgets produces trivial/low/moderate/severe/extreme thresholds by party size
- [x] **XP-05**: calculateEncounterRating maps total XP to threat rating string
- [x] **XP-06**: calculateXP orchestrates creatures + hazards + party into final XP total and threat rating

### Damage Taxonomy

- [x] **DMG-01**: Physical, energy, and all damage type constants with category mapping
- [x] **DMG-02**: Material damage effects, die sizes, and die face constants
- [x] **DMG-03**: TypeScript interfaces for damage formula, base damage, IWR bypass, and critical inclusion
- [x] **DMG-04**: DamageCategorization utility maps damage types to/from categories
- [x] **DMG-05**: nextDamageDieSize steps die size up/down (d4-d6-d8-d10-d12) with capping

### Modifiers & Stacking

- [x] **MOD-01**: Modifier class with slug, label, value, type (7 PF2e types), enabled state
- [x] **MOD-02**: applyStackingRules applies PF2e stacking (highest bonus/lowest penalty per type, untyped all stack)
- [x] **MOD-03**: StatisticModifier aggregates modifier list into totalModifier via stacking rules
- [x] **MOD-04**: DamageDicePF2e holds dice modifier with diceNumber, dieSize, damageType, category, critical

### IWR Engine

- [x] **IWR-01**: IWR data models for Immunity, Weakness, Resistance with type, value, exceptions
- [x] **IWR-02**: applyIWR applies immunities then weaknesses then resistances to damage instances
- [x] **IWR-03**: Critical-hits immunity undoubles crit damage, precision immunity reduces precision component
- [x] **IWR-04**: Weakness doubleVs and applyOnce special cases handled correctly

### Conditions

- [x] **COND-01**: CONDITION_SLUGS (44 conditions), VALUED_CONDITIONS subset, CONDITION_GROUPS (detection/attitudes)
- [x] **COND-02**: ConditionManager add/remove/has/get conditions with numeric value tracking
- [x] **COND-03**: Dying removal auto-increments wounded; mutually exclusive group enforcement

## Future Requirements

### Weapon Damage Calculator (deferred from v2.0)

- **WDMG-01**: WeaponDamagePF2e calculates full damage formula with striking runes, property runes, specialization
- **WDMG-02**: Rune data integration for weapon damage bonuses

### UI Integration (next milestone)

- **UI-01**: Encounter builder UI wired to XP budget engine
- **UI-02**: Damage calculator UI with IWR application display
- **UI-03**: Condition tracker integrated with ConditionManager

## Out of Scope

| Feature | Reason |
|---------|--------|
| Foundry VTT Roll system | Use simple dice library instead; Foundry Roll class too coupled |
| Rules engine (RuleElement, Predicates) | 50+ file subsystem, too complex for standalone port |
| i18n / localization | English strings directly; no game.i18n dependency |
| UI layer for game logic | Engine-only milestone; UI wiring deferred to next milestone |
| Weapon damage calculator | Deep coupling to rune data and rules engine; creature damage already in JSON |
| Foundry Canvas/Actor/Item classes | Framework code, not game logic |
