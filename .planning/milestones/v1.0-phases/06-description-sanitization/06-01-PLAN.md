  <behavior>
    ## DESC-01: @UUID link parsing

    - @UUID with label (pf2e compendium):
      Input: `@UUID[Compendium.pf2e.spells-srd.Item.abc123]{Fireball}`
      Output: `<a data-entity-pack="spells-srd" data-entity-id="abc123" class="pf2e-link cursor-pointer text-blue-600 underline hover:text-blue-800 font-medium">Fireball</a>`

    - @UUID without label (pf2e compendium):
      Input: `@UUID[Compendium.pf2e.spells-srd.Item.abc123]`
      Output: `<a data-entity-pack="spells-srd" data-entity-id="abc123" class="pf2e-link cursor-pointer text-blue-600 underline hover:text-blue-800 font-medium">[abc123]</a>`

    - @UUID non-pf2e compendium with label:
      Input: `@UUID[Compendium.dnd5e.monsters.Item.xyz]{Dragon}`
      Output: `Dragon` (plain text fallback — label preserved, wrapper stripped)

    - @UUID non-pf2e compendium without label:
      Input: `@UUID[Compendium.dnd5e.monsters.Item.xyz]`
      Output: `Compendium.dnd5e.monsters.Item.xyz` (bracket content as plain text)

    ## DESC-02: Inline element rendering

    - @Damage with damage type:
      Input: `@Damage[2d6[fire]]`
      Output: `<span class="pf2e-damage font-bold text-orange-600">2d6 fire</span>`

    - @Damage with unknown type:
      Input: `@Damage[1d4[sonic]]`
      Output: `<span class="pf2e-damage font-bold text-gray-800">1d4 sonic</span>` (default color)

    - @Damage with pipe-separated options:
      Input: `@Damage[2d6[fire]|persistent]`
      Output: `<span class="pf2e-damage font-bold text-orange-600">2d6 fire</span>` (options after pipe ignored in display)

    - @Check with basic and DC:
      Input: `@Check[reflex|dc:25|basic]`
      Output: `<span class="pf2e-check font-bold text-gray-900">basic reflex DC 25</span>`

    - @Check with only type:
      Input: `@Check[fortitude]`
      Output: `<span class="pf2e-check font-bold text-gray-900">fortitude</span>`

    - @Template with distance:
      Input: `@Template[cone|distance:60]`
      Output: `<span class="pf2e-template font-bold text-gray-800">cone 60 ft.</span>`

    - @Template without distance:
      Input: `@Template[emanation]`
      Output: `<span class="pf2e-template font-bold text-gray-800">emanation</span>`

    ## Fallback and edge cases

    - Unrecognized @-syntax with label:
      Input: `@Localize[PF2E.SomeKey]{Localized Text}`
      Output: `Localized Text`

    - Unrecognized @-syntax without label:
      Input: `@Action[strike]`
      Output: `strike`

    - Standard HTML passed through unchanged:
      Input: `<p><strong>Bold</strong> text</p>`
      Output: `<p><strong>Bold</strong> text</p>`

    - Malformed @-syntax (unclosed bracket):
      Input: `@UUID[malformed`
      Output: `@UUID[malformed` (unchanged)

    - Empty string:
      Input: ``
      Output: ``

    - Mixed content (HTML + multiple @-syntax tokens):
      Input: `<p>Deal @Damage[2d6[fire]] damage, then attempt a @Check[reflex|dc:25|basic] save.</p>`
      Output: `<p>Deal <span class="pf2e-damage font-bold text-orange-600">2d6 fire</span> damage, then attempt a <span class="pf2e-check font-bold text-gray-900">basic reflex DC 25</span> save.</p>`

    - Labeled @UUID processed before unlabeled (regex ordering):
      Input: `@UUID[Compendium.pf2e.spells-srd.Item.abc]{Fireball}`
      Output must contain `>Fireball<` and must NOT contain `{Fireball}`

    2. Export `sanitizeDescription(html: string): string` — chains 7 `.replace()` calls in this exact order:
       - Pass 1: `@UUID[Compendium.pf2e.PACK.Item.ID]{Label}` -> `<a>` with data attributes, pf2e-link class, Tailwind blue link styling
         Regex: `/@UUID\[Compendium\.pf2e\.([^.\]]+)\.Item\.([^\]]+)\]\{([^}]+)\}/g`
       - Pass 2: `@UUID[Compendium.pf2e.PACK.Item.ID]` (no label) -> `<a>` with `[ID]` as text
         Regex: `/@UUID\[Compendium\.pf2e\.([^.\]]+)\.Item\.([^\]]+)\]/g`
       - Pass 3: `@Damage[expr]` -> `<span class="pf2e-damage font-bold {colorClass}">cleaned</span>`
         Regex: `/@Damage\[([^\]]+)\]/g`
         Parse: split on `|`, take first part, replace inner `[type]` with ` type`, lookup color
       - Pass 4: `@Check[expr]` -> `<span class="pf2e-check font-bold text-gray-900">text</span>`
         Regex: `/@Check\[([^\]]+)\]/g`
         Parse: split on `|`, extract type, dc:N, basic flag
       - Pass 5: `@Template[expr]` -> `<span class="pf2e-template font-bold text-gray-800">text</span>`
         Regex: `/@Template\[([^\]]+)\]/g`
         Parse: split on `|`, extract type, distance:N
       - Pass 6: `@Type[...]{Label}` fallback -> keep Label only
         Regex: `/@\w+\[[^\]]*\]\{([^}]+)\}/g`
       - Pass 7: `@Type[content]` fallback -> keep content only
         Regex: `/@\w+\[([^\]]*)\]/g`

    CRITICAL ordering: Pass 1 (labeled @UUID) MUST run before Pass 2 (unlabeled @UUID). Passes 6-7 (fallback) MUST be last.
