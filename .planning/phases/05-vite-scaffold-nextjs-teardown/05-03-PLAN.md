**Step 2: Create `eslint.config.js` at project root.**

Per D-08: All rules enabled as errors, not warnings. Per D-09: Include BOTH `eslint-plugin-boundaries` (FSD layer direction) AND `eslint-plugin-import` (import hygiene, no circular deps). Both packages are installed in devDependencies (added in Plan 05-01). `eslint-import-resolver-typescript` is also installed to resolve `@/*` and `@engine` path aliases.

Exact eslint-plugin-import rule selection is Claude's discretion per CONTEXT.md. Minimal config with `import/no-cycle` and `import/no-duplicates` satisfies D-09.

```javascript
import boundaries from 'eslint-plugin-boundaries'
import importPlugin from 'eslint-plugin-import'

export default [
  {
    files: ['src/**/*.{ts,tsx}'],
    plugins: {
      boundaries,
      import: importPlugin,
    },
    settings: {
      'boundaries/elements': [
        { type: 'app',      pattern: ['src/app/**'],      mode: 'full' },
        { type: 'pages',    pattern: ['src/pages/**'],    mode: 'full' },
        { type: 'widgets',  pattern: ['src/widgets/**'],  mode: 'full' },
        { type: 'features', pattern: ['src/features/**'], mode: 'full' },
        { type: 'entities', pattern: ['src/entities/**'], mode: 'full' },
        { type: 'shared',   pattern: ['src/shared/**'],   mode: 'full' },
      ],
      'import/resolver': {
        typescript: {
          alwaysTryTypes: true,
          project: './tsconfig.json',
        },
      },
    },
    rules: {
      // FSD layer import direction (D-09: eslint-plugin-boundaries)
      'boundaries/element-types': ['error', {
        default: 'disallow',
        rules: [
          { from: 'app',      allow: ['pages', 'widgets', 'features', 'entities', 'shared'] },
          { from: 'pages',    allow: ['widgets', 'features', 'entities', 'shared'] },
          { from: 'widgets',  allow: ['features', 'entities', 'shared'] },
          { from: 'features', allow: ['entities', 'shared'] },
          { from: 'entities', allow: ['shared'] },
          { from: 'shared',   allow: [] },
        ],
      }],

      // Import hygiene (D-09: eslint-plugin-import)
      'import/no-cycle': 'error',
      'import/no-duplicates': 'error',
      'import/no-self-import': 'error',
    },
  },
  {
    ignores: ['node_modules/**', 'dist/**', 'src-tauri/**', 'engine/**'],
  },
]
```

NOTE: If `eslint-plugin-import` v2 has issues with ESLint 10 flat config, the executor should check if the `eslint-plugin-import` package installed supports flat config. If not, try wrapping with `@eslint/compat`'s `fixupPluginRules()`, or replace with the drop-in fork `eslint-plugin-import-x` (which has native flat config support) — the user locked `eslint-plugin-import` semantics, not a specific npm package name. Either way, the three rules (`no-cycle`, `no-duplicates`, `no-self-import`) MUST be present as errors.
