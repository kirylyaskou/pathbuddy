## Spacing Scale

Standard 8-point scale — all values are multiples of 4px. All Tailwind spacing classes map to this scale (gap-1=4px, gap-2=8px, gap-4=16px, gap-6=24px, gap-8=32px, gap-12=48px, gap-16=64px).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, stat-cell inner spacing, rank badge gaps |
| sm | 8px | Between skill rows, between stat chips, tab trigger gaps |
| md | 16px | Sheet content horizontal padding, section vertical padding |
| lg | 24px | Between major sections (Core block → Skills block) |
| xl | 32px | Between tab content top and first section |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Compact stat grid (HP/AC/Speed row): 8px column gap between chips, 4px inner padding
- Tab list: no vertical padding inside TabsList, 2px indicator underline
- Skill row height: fixed 28px (compact density for long lists)
- DM Notes textarea: 8px padding, min-height 200px

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Skill values, spell lists, feat descriptions, equipment rows |
| Micro | 12px | 400 | 1.2 | Rank badge letters (T/E/M/L/U), sub-labels under stat chips, feat meta lines, class info line |
| Heading | 16px | 600 | 1.2 | Section headings (Core Stats, Skills, Armor, Weapons, etc.) |
| Display | 20px | 600 | 1.2 | Character name in SheetHeader (SheetTitle) |

Two weights only: **400** (regular) and **600** (semibold). Weight 500 is not used.

Usage notes (no additional sizes):
- Stat chip values (HP/AC/Speed numbers): 16px / 600 — Heading weight makes numbers stand out from 400-weight labels
- Ability score values: 16px / 600 (same tier as section headings — context separates them visually)
- Ability modifier sub-label: 12px / 400 (Micro)
- Modifier with sign (+7, -1) in skill column: 14px / 400 (Body)
- Class info line ("Fighter · 5 · Human"): 12px / 400 / muted-foreground (Micro)

## Color

All values reference CSS custom properties from `src/app/styles/globals.css`. Dark mode tokens are declared in `.dark {}` and activate automatically.

| Role | Token | Light value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | oklch(0.96 0.015 70) | Sheet panel background, page background |
| Secondary (30%) | `--card` | oklch(0.98 0.012 65) | Stat chip cards, section container backgrounds |
| Accent (10%) | `--primary` | oklch(0.55 0.14 65) | Active tab indicator, SheetTitle text color |
| Border | `--border` | oklch(0.82 0.03 60) | Section dividers, stat chip borders, skill row separators |
| Muted text | `--muted-foreground` | oklch(0.45 0.03 55) | Ability modifier sub-labels, class info line, placeholder text |
| Destructive | `--destructive` | oklch(0.50 0.20 25) | Not used in this phase |
| Gold highlight | `--pf-gold` | oklch(0.62 0.15 70) | Legendary (L) rank badge text/bg tint |
| Blood | `--pf-blood` | oklch(0.45 0.18 25) | Not used in this phase |

Accent reserved for: active tab underline indicator, SheetTitle character name, active TabsTrigger text.
Never apply accent to: skill rank badges, equipment text, body copy, icons.

### Proficiency Rank Badge Colors

Rank badges use `bg-{color}/15 text-{color}` pattern (15% opacity background tint):

| Rank | Letter | Token | Color |
|------|--------|-------|-------|
| Untrained | U | `muted` / `muted-foreground` | Grey — `bg-muted text-muted-foreground` |
| Trained | T | `--pf-threat-low` oklch(0.50 0.12 145) | Green — `bg-pf-threat-low/15 text-pf-threat-low` |
| Expert | E | `--pf-skill-expert` oklch(0.50 0.16 235) | Blue — NEW TOKEN — add to globals.css |
| Master | M | `--pf-rarity-rare` oklch(0.50 0.18 285) | Purple — `bg-pf-rarity-rare/15 text-pf-rarity-rare` |
| Legendary | L | `--pf-gold` oklch(0.62 0.15 70) | Amber/gold — `bg-pf-gold/15 text-pf-gold` |

New token to add to both `:root` and `.dark` blocks in `globals.css`:
```css
/* :root */
--pf-skill-expert: oklch(0.50 0.16 235);
/* .dark */
--pf-skill-expert: oklch(0.60 0.18 235);
```
And in `@theme inline`:
```css
--color-pf-skill-expert: var(--pf-skill-expert);
```

### Stat Chips (Core Stats row)

Three chips in a horizontal row: **HP · AC · Speed**

| Property | Value |
|----------|-------|
| Container | `flex gap-2 px-4 py-3` |
| Chip | `rounded-md border border-border bg-card px-3 py-2 flex flex-col items-center` |
| Value | 16px / weight 600 / foreground |
| Label | 12px / weight 400 / muted-foreground (e.g. "HP", "AC", "Speed") |
| HP chip special | show "N / N" format (current=max at load; current HP not tracked in this phase) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None — panel is read-only (DM Notes saves silently on blur) |
| Panel title | `{character.name}` — pulled directly from character data |
| Sub-title | `{class} · {level} · {ancestry}` (e.g. "Fighter · 5 · Human") |
| Skills section | "Skills" |
| Core stats section | "Core Stats" (or no label — chips are self-labeling) |
| Empty state — spells | "No spellcasting" (centered, muted-foreground, 14px) |
| Empty state — feats | "No feats recorded" (centered, muted-foreground, 14px) |
| Empty state — equipment | "No equipment recorded" (centered, muted-foreground, 14px) |
| Notes placeholder | "DM notes for this character..." |
| HP label | "HP" |
| AC label | "AC" |
| Speed label | "Speed" |
| Perception label | "Perception" |
| Saves labels | "Fortitude" / "Reflex" / "Will" (full names, abbreviated to "Fort" / "Ref" / "Will" if space is tight) |
| Ability abbreviations | STR · DEX · CON · INT · WIS · CHA |
| Lore skill suffix | "{name} Lore" — preserve as-is from data |
| Rune notation format | "{name} +{pot}" + " [{res}]" if res non-empty + " [{rune1}, ...]" if runes non-empty |
| Error state — parse failure | "Could not read character data." (centered, muted-foreground, 14px) — shown inside Sheet when JSON.parse(rawJson) throws; Sheet remains open with this message in place of tab content |
| Destructive actions | None in this phase |
