## Spacing Scale

Declared values (all multiples of 4, mapped to Tailwind utility classes):

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge inline padding, condition pill padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, button padding (short), HP display gap |
| md | 16px | `p-4` / `gap-4` | Default card padding, form field spacing |
| lg | 24px | `p-6` / `gap-6` | Section padding, modal padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major sections |
| 2xl | 48px | `py-12` | Empty state vertical padding |
| 3xl | 64px | `py-16` | Full-page vertical centering offset |

**Exceptions:**

- Sidebar width: 180px (fixed, not a spacing token — use `w-[180px]`)
- Creature card left border: `border-l-4` (4px — xs) — active state, inactive state
- HP bar height: 6px (`h-1.5`) — 4px (`h-1`) is too thin to read HP color thresholds at a glance; 8px (`h-2`) is too tall and visually dominates the compact card row. 6px sits between both and renders the three color zones (gold/amber/crimson) clearly on a 128px wide track.
- Initiative badge: 32px × 32px (`w-8 h-8`) — circular, centered number
- Touch/click targets minimum: 32px height (`min-h-8`) for all interactive elements

## Typography

| Role | Font | Size | Tailwind | Weight | Tailwind | Line Height |
|------|------|------|----------|--------|----------|-------------|
| Body | system-ui | 14px | `text-sm` | 400 Regular | `font-normal` | 1.5 |
| Label | system-ui | 12px | `text-xs` | 400 Regular | `font-normal` | 1.4 |
| Heading | Cinzel | 20px | `text-xl` | 700 Bold | `font-bold` | 1.2 |
| Display | Cinzel | 28px | `text-3xl` | 700 Bold | `font-bold` | 1.1 |

**Application rules:**

- `font-display` (Cinzel) is used ONLY for: page section headings, creature name on card, sidebar "Pathbuddy" brand title, modal/dialog headings, stat block panel title
- `font-body` (system-ui) is used for: all body copy, form labels, badge text, condition names, HP numbers, nav item labels, sync stage labels, error messages
- No italic. No font sizes outside the 4 declared above.
- All text on dark backgrounds: minimum contrast ratio 4.5:1 (WCAG AA)
  - On charcoal-800 (`#1a1a1a`): body text uses `text-stone-200` (`#e7e5e4`) — ratio ≈ 12:1
  - Muted/secondary text: `text-stone-400` (`#a8a29e`) — ratio ≈ 5.5:1
  - Gold accent text: `text-gold` (`#c9a84c`) on charcoal-800 — ratio ≈ 5.2:1

### ConditionBadge (replaces ConditionToggle)

Each condition on a creature card renders as an inline pill badge:

- Size: `px-2 py-0.5 rounded-full text-xs font-normal font-body`
- Click: toggles condition; if condition has a numeric value, opens inline value editor
- Conditions with values (stunned, slowed, frightened, dying): badge shows `{Name} {value}` — e.g., "Stunned 2"
  - Click increments value; shift+click or right-click decrements; when value reaches 0, condition is removed
  - Alternate UX: badge click opens a tiny +/- popover (3-button: `−`, value display, `+`) adjacent to badge
- Conditions without values (prone, flat-footed, invisible, unconscious, etc.): badge shows name only
- Click any badge to toggle condition off (remove it from the creature)
- Inactive conditions (not applied) are NOT shown on the card — only active conditions are rendered
- "Add condition" affordance: small `+` icon button after the last badge, `w-6 h-6 rounded-full bg-charcoal-500 hover:bg-charcoal-400 text-stone-400 hover:text-stone-200 text-xs flex items-center justify-center` — opens a condition picker popover listing all CONDITION_DEFS

**Condition picker popover:**
- `absolute bg-charcoal-600 border border-charcoal-500 rounded-lg p-2 shadow-card z-10`
- All 11 conditions listed as small toggleable pills in 2 columns
- Already-active conditions shown in their active color (dimmed) — clicking removes them
- Click outside to close (no confirm step)

## Interaction Contracts

### HP Modification Flow

1. User sets `amount` in the amount input (default: 1, range: 1–999)
2. Clicks `−` → `emit('modifyHp', -amount)` → store subtracts from `currentHP`
3. Clicks `+` → `emit('modifyHp', +amount)` → store adds to `currentHP`
4. `currentHP` is clamped to `[0, maxHP]` by the store (existing behavior, no UI change)
5. HP bar fill width and color update reactively on `currentHP` change
6. Amount input value persists; user clears manually or types a new value

### Condition Value Flow

For conditions that carry a numeric value (stunned N, slowed N, etc.):

1. Condition not active → click "Add Condition" `+` button → condition picker popover opens
2. Select a value-bearing condition → it activates with default value `1`
3. Badge shows "Stunned 1" — click badge increment button to increase, decrement to decrease to 0 (removes)
4. OR: shift+click badge to decrement directly without opening popover
5. Binary conditions (prone, flat-footed, etc.) → toggle directly on click (no value)

### Drag-and-Drop (unchanged behavior, reskinned)

- Draggable: creature card when `currentHP > 0 && !isCurrentTurn`
- Drag-over target: gold ring highlight instead of blue ring
- Drop: `combatStore.reorderCreature()` (no change)

### Navigation

- `RouterLink` active class: Vue Router `router-link-active` and `router-link-exact-active` both receive the gold sidebar treatment
- Programmatic navigation: not introduced in this phase
- Splash → app: existing 300ms delay, no change
