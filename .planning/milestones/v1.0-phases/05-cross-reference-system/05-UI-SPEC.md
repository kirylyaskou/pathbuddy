## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, condition chip gaps |
| sm | 8px | Compact element spacing within stat block rows |
| md | 16px | Default element spacing, section padding, card padding |
| lg | 24px | Section heading padding, panel header padding |
| xl | 32px | Layout gaps between stat block groups |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Not used in this phase |

Exceptions: Panel close button touch target: 40px × 40px (nearest multiple of 8, sufficient for desktop mouse target).

**Source:** RESEARCH.md established Tailwind utility conventions. Existing components use `p-4` (16px), `gap-2/3/4` (8/12/16px), `py-6` (24px), `py-12` (48px) — matches this scale exactly.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 14px (text-sm) | 700 (bold) | 1.4 |
| Heading | 18px (text-lg) | 700 (bold) | 1.2 |
| Panel Title | 20px (text-xl) | 700 (bold) | 1.2 |

**Weights declared:** 400 (normal) and 700 (bold) only. No semibold (600) anywhere in this phase.

**Notes:**
- Body (14px/400/1.5): stat block row descriptions, expanded ability text, metadata lines (saves, HP values)
- Label (14px/700/1.4): stat block row names (clickable item names), section group headers (e.g., "Melee Strikes", "Spells"), stat value labels (HP, AC, Saves)
- Heading (18px/700/1.2): creature name at top of panel; canonical entity name when navigated
- Panel Title (20px/700/1.2): panel header displaying the currently viewed creature/entity name

**Source:** Existing codebase uses `text-2xl font-bold` (24px) for page headings, `text-lg font-bold` (18px) for creature card names, `text-sm` (14px) for metadata — this phase's panel follows the same scale compressed for the compact stat block format. Panel title at 20px matches the round counter (`text-xl`) from CombatTracker, staying below the 24px page heading.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f3f4f6` (gray-100) | Panel backdrop overlay tint, stat block row default background |
| Secondary (30%) | `#ffffff` (white) | Panel surface, stat block section backgrounds, expanded description areas |
| Accent — Canonical (10%) | `#3b82f6` (blue-500) | Canonical item left border (`border-l-4`), canonical item row background (`bg-blue-50`), link icon color |
| Accent — NPC-Unique | `#f59e0b` (amber-500) | NPC-unique item left border (`border-l-4`), NPC-unique item row background (`bg-amber-50`) |
| Destructive | `#dc2626` (red-600) | Not used in this phase — no destructive actions |

**Accent reserved for:**
- `border-l-4 border-blue-500` on canonical item rows (left border stripe only)
- `bg-blue-50` background tint on canonical item rows
- Link icon SVG fill/stroke on canonical items (the clickable navigate-to-canonical affordance)
- `border-l-4 border-amber-500` on NPC-unique item rows (left border stripe only)
- `bg-amber-50` background tint on NPC-unique item rows
- Back button hover state: `hover:text-blue-600`

**Backdrop overlay:** `bg-black/30` at z-40. Clicking backdrop closes panel.

**Panel shadow:** `shadow-xl` (matches SplashScreen and DashboardView card pattern).

**Source:**
- CONTEXT.md (locked): "Canonical items: colored left border/badge (e.g., blue)"; "NPC-unique abilities: different colored border (e.g., amber/gold)"
- RESEARCH.md (recommended values): `border-l-4 border-blue-500 bg-blue-50` for canonical; `border-l-4 border-amber-500 bg-amber-50` for NPC-unique
- Existing codebase: `border-l-4 border-blue-500 bg-blue-50` already in use on `CreatureCard.vue` for current-turn state — exact same classes, consistent visual language

## Focal Point

The **creature/entity name in the panel header** is the primary visual anchor of `CreatureDetailPanel`. It is rendered at Panel Title size (20px/700) and remains visible at all times via the sticky header. Every navigation action — opening the panel, navigating to a canonical entity, or pressing back — updates this name, making it the single orientation cue for the user at all times.

## Component Inventory

### New: `CreatureDetailPanel.vue`

Fixed-position slide-over panel anchored to the right edge of the screen.

**Layout:**
- Outer: `fixed right-0 top-0 h-full w-96 z-50 bg-white shadow-xl overflow-y-auto flex flex-col`
- Panel width: 384px (`w-96`) — fits alongside the 6xl max-width combat tracker on desktop
- Backdrop: `fixed inset-0 z-40 bg-black/30` — click to close

**Panel header (sticky):**
- `sticky top-0 bg-white border-b border-gray-200 px-4 py-3 flex items-center gap-3`
- Back button (shown only when `navigationHistory.length > 0`): `← Back` text, `text-blue-600 hover:text-blue-700 text-sm font-bold`
- Creature/entity name: Panel Title typography (20px/bold) — **focal point**
- Close button (×): top-right, 40×40px target, `text-gray-400 hover:text-gray-600`

**Stats row (top of stat block):**
- Horizontal flex row: `flex gap-4 px-4 py-3 bg-gray-50 border-b border-gray-100`
- Each stat: label above value — label at Label typography (14px/bold), value at Heading size (18px/bold)
- Stats displayed: HP (current/max), AC, Perception, Fort/Ref/Will saves
- Format: `HP 85/85`, `AC 27`, `Perc +18`, `Fort +15 / Ref +12 / Will +18`

**Section groups:**
- Section header: `px-4 py-2 text-xs font-bold uppercase tracking-wide text-gray-500 bg-gray-50 border-b border-gray-100`
- Section order: Stats (always first), Melee Strikes, Ranged Strikes, Spellcasting, Actions/Feats, Equipment, Other
- Empty sections are omitted entirely

**Item rows (collapsed):**
- `flex items-center gap-2 px-4 py-2 cursor-pointer hover:bg-gray-50 border-b border-gray-100`
- Left border stripe: `border-l-4` in canonical blue or NPC-unique amber
- Row background: `bg-blue-50` or `bg-amber-50`
- Item name: Label typography (14px/bold), left-aligned
- Link icon (canonical only): inline SVG, 16×16px, `text-blue-500`, right-aligned, triggers `navigateToCanonical`
- Expand chevron: inline SVG arrow, 16×16px, `text-gray-400`, rotates 90deg when expanded

**Item rows (expanded):**
- `px-4 py-3 border-b border-gray-100` + same background tint as collapsed state
- Description text: Body typography (14px/400/1.5)
- Long descriptions: no truncation — panel scrolls; description renders in full
- `v-html` used to render description (rawData `system.description.value` is HTML)

**Loading state (resolving items):**
- Panel shows creature name in header immediately
- Stats row renders immediately from `rawData.system.attributes`
- Item rows show skeleton placeholder: 3 rows of `animate-pulse bg-gray-200 h-4 rounded` while `isResolving = true`

**Animation:**

```css
/* Panel slide */
.panel-slide-enter-active,
.panel-slide-leave-active {
  transition: transform 250ms ease;
}
.panel-slide-enter-from,
.panel-slide-leave-to {
  transform: translateX(100%);
}

/* Backdrop fade */
.panel-backdrop-enter-active,
.panel-backdrop-leave-active {
  transition: opacity 200ms ease;
}
.panel-backdrop-enter-from,
.panel-backdrop-leave-to {
  opacity: 0;
}

/* Expand chevron */
.chevron-expanded {
  transform: rotate(90deg);
  transition: transform 150ms ease;
}
```

**Source:** RESEARCH.md Pattern 3 (slide-over component sketch); CONTEXT.md locked decisions (slide-over, inline expand, compact card).

## Interaction Contract

### Opening the panel
- User clicks "View Stat Block" on a `CreatureCard` that has `creature.sourceId` set
- Panel slides in from right in 250ms
- Backdrop fades in at 200ms
- Panel shows creature name immediately; stats and items load asynchronously
- Combat tracker remains visible and interactive behind the backdrop

### Closing the panel
- Click backdrop: panel slides out, backdrop fades
- Click × button: same animation
- Navigation back beyond root: not possible (back button hidden at root)

### Navigating to a canonical entity
- User clicks link icon on a canonical item row
- Current view pushed to `navigationHistory` stack
- Panel content replaces with the canonical entity's stat block
- Scroll position resets to top
- Back button appears in header

### Navigating back
- User clicks "← Back"
- Previous view popped from `navigationHistory`
- Panel content replaces with previous view
- Scroll position resets to top
- Back button hidden again if history is now empty

### Expanding an item
- User clicks an item row (anywhere except the link icon)
- Row expands inline to show full description
- Chevron rotates 90deg in 150ms
- Clicking again collapses (toggle behavior)
- Multiple items can be expanded simultaneously

### Expanded item state key
- Tracked as `Set<string>` keyed on `item.embedded._id`
- Persists within a single panel open session
- Cleared on `close()` or navigation
