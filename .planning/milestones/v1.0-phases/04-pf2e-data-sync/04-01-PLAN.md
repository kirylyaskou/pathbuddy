  <action>
Create `src-tauri/src/commands.rs` with these six `#[tauri::command]` functions:

1. **`download_file(url: String, dest_path: String) -> Result<(), String>`**
   - Use `reqwest::get(&url).await` which follows up to 10 redirects automatically (required for GitHub zipball_url 302->S3 redirect chain)
   - Call `.bytes().await` to get full response body
   - Write bytes to `dest_path` via `std::fs::File::create` + `file.write_all`
   - Map all errors to `String` via `.map_err(|e| e.to_string())?`

2. **`extract_zip(zip_path: String, dest_path: String) -> Result<(), String>`**
   - Open zip file with `std::fs::File::open(&zip_path)`
   - Create `zip::ZipArchive::new(file)`
   - Call `archive.extract(&dest_path)` which handles nested directories
   - Map all errors to String

3. **`glob_json_files(base_path: String) -> Result<Vec<String>, String>`**
   - Recursively walk `base_path` directory using a helper function `collect_json_files(dir: &Path, results: &mut Vec<String>)`
   - For each entry: if directory, recurse; if file with `.json` extension, add to results
   - CRITICAL: Normalize all path separators to forward slashes before returning. Use `path.to_string_lossy().replace('\\', '/')` to ensure TypeScript receives consistent `/`-separated paths on Windows and Unix
   - The `_pattern` parameter from plan.txt is NOT needed; remove it. The glob is done by the TS layer filtering the returned file list

4. **`read_text_file(path: String) -> Result<String, String>`**
   - Simple `std::fs::read_to_string(&path).map_err(|e| e.to_string())`

5. **`remove_dir(path: String) -> Result<(), String>`**
   - `std::fs::remove_dir_all(&path).map_err(|e| e.to_string())`

6. **`remove_file(path: String) -> Result<(), String>`**
   - `std::fs::remove_file(&path).map_err(|e| e.to_string())`

Required imports at top of file:
```rust
use std::fs;
use std::io::Write;
use std::path::Path;
use tauri::command;
```

All six functions must be `pub` and `async` (Tauri 2 command convention).

**lib.rs changes:**

Add `mod commands;` declaration at the top of the file (before the `pub fn run()` function).

Add `.invoke_handler()` call to the Tauri Builder chain, BEFORE `.run()`. Register all six commands in a single `generate_handler![]` macro call (CRITICAL: only one invoke_handler call allowed — last one wins):

```rust
mod commands;

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_sql::Builder::default().build())
        .plugin(tauri_plugin_http::init())
        .plugin(tauri_plugin_fs::init())
        .invoke_handler(tauri::generate_handler![
            commands::download_file,
            commands::extract_zip,
            commands::glob_json_files,
            commands::read_text_file,
            commands::remove_dir,
            commands::remove_file,
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```
