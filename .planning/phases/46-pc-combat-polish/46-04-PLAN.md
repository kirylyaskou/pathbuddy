## Objective

Two independent polish tasks:

1. **EncounterCreatureSearchPanel creature type filter** — add a creature type `Select` (shadcn)
   and switch the component from `searchCreatures` / `fetchCreatures` to `searchCreaturesFiltered`.

2. **CharactersPage encounter picker** — when >1 encounter tabs are open and the user clicks
   "Add to Combat" on a character, show an encounter tab picker instead of adding directly.
   Uses `useEncounterTabsStore` to get open tabs and `addCombatantToTab` to route the combatant.

## Context

- **EncounterCreatureSearchPanel**: currently uses `searchCreatures(query, 50)` / `fetchCreatures(50, 0)`.
  Switch to `searchCreaturesFiltered({ query, traits: [type] }, 50)`. Add `CREATURE_TYPES` const
  (same 20-entry array as Plan 46-02). Add `creatureType` state. Place the type `Select` after the
  search Input. Use `text-xs h-7` for compact trigger — matches search input row height.
- **CharactersPage encounter picker**: current `handleAddToCombat` calls `addCombatant(combatant)`
  directly. New logic:
  - Read `openTabs` from `useEncounterTabsStore.getState()` at call time (no hook needed — just
    `.getState()` in the event handler to avoid extra renders).
  - 0 or 1 tabs: existing behavior unchanged (direct `addCombatant` for 0 tabs, or
    `addCombatantToTab(openTabs[0].id, combatant)` for 1 tab — same effect since that tab is active).
  - >1 tabs: store the pending combatant in state and show a picker Dialog.
  - Picker Dialog: shadcn `Dialog` with title "Add to Encounter" and a list of encounter tab names
    as buttons. Clicking one calls `addCombatantToTab(tabId, combatant)` + shows toast + closes dialog.
  - Add state: `pendingCombatant: Combatant | null`, `pickerOpen: boolean`.
  - The combatant creation logic (lines 44–58 of CharactersPage) is unchanged.

```typescript
function handleAddToCombat(character: CharacterRecord) {
  try {
    const build = JSON.parse(character.rawJson) as PathbuilderBuild
    const maxHp = calculatePCMaxHP(build)
    const ac = build.acTotal.acProfBonus + build.acTotal.acAbilityBonus + build.acTotal.acItemBonus
    const combatant: Combatant = {
      id: crypto.randomUUID(),
      creatureRef: character.id,
      displayName: character.name,
      initiative: 0,
      hp: maxHp,
      maxHp,
      tempHp: 0,
      isNPC: false,
      ac,
    }

    const openTabs = useEncounterTabsStore.getState().openTabs

    if (openTabs.length > 1) {
      setPendingCombatant(combatant)
      setPickerOpen(true)
      return
    }

    if (openTabs.length === 1) {
      useEncounterTabsStore.getState().addCombatantToTab(openTabs[0].id, combatant)
    } else {
      addCombatant(combatant)
    }
    toast(`${character.name} added to combat`, {
      action: { label: 'Go to Combat', onClick: () => navigate('/combat') },
    })
  } catch {
    toast(`Failed to add ${character.name} to combat`)
  }
}
```
