## Spacing Scale

Declared values (all multiples of 4). Source: established Tailwind usage across existing components.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), button padding (`py-1`) |
| sm | 8px | Badge row gaps (`gap-2`), button padding (`px-2 py-1`) |
| md | 16px | Card padding (`p-4`), section internal spacing |
| lg | 24px | Column gaps inside creature card (`gap-4` = 16px; use 24px for picker sections) |
| xl | 32px | Picker popover width minimum (264px = 33× 8px scale) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Condition badge pill: `py-0.5` (2px vertical padding) — retained from existing ConditionBadge.vue; the extra 2px gives correct pill aspect ratio at `text-xs`; visual density would break at `py-1` (4px)
- Condition picker popover: `w-64` (256px fixed) — established in ConditionBadge.vue, retained
- Add-condition button: `w-6 h-6` (24px) circular touch target — retained from existing component
- Initiative badge: `w-8 h-8` (32px) circular — retained from CreatureCard.vue

## Color

Source: tailwind.config.js custom palette + CreatureCard.vue + AppLayout.vue.

Primary focal point: the active-turn creature name rendered in `text-gold font-display font-bold` is the visual anchor of the combat tracker. The condition badge row beneath it is the secondary focal point.

| Role | Tailwind Token | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | `charcoal-800` | `#1a1a1a` | App background (`bg-charcoal-800`) |
| Secondary (30%) | `charcoal-600` | `#2a2a2a` | Creature cards, sidebar, picker popover (`bg-charcoal-600`) |
| Accent (10%) | `gold` | `#c9a84c` | Reserved for specific elements only — see list below |
| Destructive | `crimson` | `#8b2020` | Debilitating condition badges only — see semantic families below |

Accent (`gold` / `#c9a84c`) is reserved for:
- Active turn creature name text
- Active turn initiative badge background
- Active turn card left border
- Active nav item left border + text
- Sidebar brand icon + brand name
- "View Stat Block" button text
- Gold glow box-shadow on active-turn card

| Family | Color Token | Conditions |

## Interaction Contracts

### ConditionBadge Picker (redesigned)

**Trigger:** Click `+` button (24px circular, `bg-charcoal-500`) adjacent to badge row.

**Popover:** `w-64` fixed width, `max-h-80` with `overflow-y-auto`, `bg-charcoal-600 border border-charcoal-500 rounded-lg p-2 shadow-card z-10`. Opens directly below trigger, left-aligned.

**Layout:** 7 vertical sections. Each section: category label row + condition buttons below in a 2-column grid.

**Category label row:** `text-[11px] font-bold text-stone-500 uppercase tracking-wider` + chevron button (`text-stone-600 hover:text-stone-400`) on the right edge. Clicking label row or chevron collapses/expands the condition grid below.

**Condition button (inactive):** `px-2 py-1 rounded text-xs text-left truncate transition-colors bg-charcoal-500 text-stone-400 hover:bg-charcoal-400 hover:text-stone-200`

**Condition button (active — already applied):** Badge class from CONDITION_BADGE_CLASSES + ` opacity-60`

**Dismiss:** Click outside popover or re-click `+` button. Uses `mousedown` listener (existing pattern, retain).

**Scroll:** Category headers are sticky within the scrollable popover: `sticky top-0 bg-charcoal-600 z-10 pb-1 pt-1`.

### Condition Badge Pill (updated)

**Display:** `px-2 py-0.5 rounded-full text-xs font-normal cursor-pointer` + badge class from CONDITION_BADGE_CLASSES.

**Valued condition counter:** Inline `<span>` after condition name — `" 2"` — no separator element, space character only.

**Click behavior on valued condition:** Decrement value by 1. When value reaches 0, call `cm.remove(slug)` which auto-removes the condition.

**Click behavior on non-valued condition:** Toggle (remove if present, which routes through `cm.remove()`).

**No hover tooltip on badge.** Title attribute retained for accessibility (screen reader only).

### Version-Counter Reactivity (implementation contract)

All `computed` properties in ConditionBadge that read from `creature.conditionManager` must list `creature.conditionVersion` as a dependency — include it in the computed body even if only as `void creature.conditionVersion`. This is the established project reactivity pattern (see STATE.md, RESEARCH.md Pattern 1).

## States

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Condition badge — active valued | Condition present + has numeric value > 1 | Badge pill with condition name + space + value number |
| Condition badge — active valued at 1 | Condition present + value is 1 | Badge pill with condition name + ` 1` (display `1` explicitly for valued conditions) |
| Condition picker — section expanded | Default on open | Full 2-col grid of condition buttons visible below category label |
| Condition picker — section collapsed | User clicks category header | Condition grid hidden; category label row remains visible; chevron rotates 180deg |
| Creature card — active turn | `isCurrentTurn === true` | Gold left border + gold-glow shadow + creature name in `text-gold` (unchanged) |
| Creature card — downed | `isDowned === true && !isCurrentTurn` | `opacity-50 border-l-4 border-crimson-dark` (unchanged) |
| Add condition button | Always visible | `w-6 h-6 rounded-full bg-charcoal-500 hover:bg-charcoal-400` (unchanged) |
