|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 16px | 600 | 1.2 |

Notes:
- Die breakdown values (individual die results, modifier, total) use JetBrains Mono at 13px/400 for alignment and scanability.
- Roll total in the toast uses 16px/600 gold (`var(--pf-gold)`) — weight distinguishes it from surrounding content, no additional size needed.

- History row formulas use 12px/400 mono; totals use 12px/600 mono.

## Interaction Contracts

### DiceCubeAnimation

- Trigger: called programmatically when `addRoll()` fires in RollStore
- Appearance: 40x40px CSS cube with 6 faces; rotates on Y axis, 360deg in 300ms ease-out, then stops on the result face
- Shows the rolled die type on each face (e.g. "d20") in 12px mono (Label size)
- Animation plays once per roll trigger — no looping
- Result face shows the rolled value in gold (`var(--pf-gold)`)
- Placement: inline next to the formula text in the toast; NOT a floating overlay
- Only visible during the toast display window; does not persist after toast dismiss
- CSS approach: `@keyframes cube-spin { 0% { transform: rotateY(0) } 100% { transform: rotateY(360deg) } }` with `animation-fill-mode: forwards` to land on result face
- No external 3D library. Uses `transform-style: preserve-3d` + 6 face divs, matching the existing `.d20` splash screen pattern in `globals.css`

### RollResultToast

- Triggered by: any `addRoll()` call to RollStore (Phase 38 will wire clickable formulas; this phase wires the toast to the store)
- Toast library: sonner (`shared/ui/sonner.tsx` already installed)
- Duration: 4 seconds, dismissible by click
- Position: bottom-right (matches existing toast usage in combat tracker)
- Content layout (top to bottom):
  1. Row 1: `[DiceCubeAnimation 40x40] [formula in Badge] [label if present, muted-foreground]`
  2. Row 2: die breakdown — `d6: [3] [5]` (each die value in a 24x24 bordered square, mono font)
  3. Row 3: modifier — `+4` or `-2` in muted-foreground mono (hidden if modifier === 0)
  4. Row 4: separator line
  5. Row 5: Total — `= 12` in 16px/600 gold (`var(--pf-gold)`)
- Natural 20 (d20 die value = 20): total shown in `var(--pf-gold)` with `.golden-glow` shadow
- Natural 1 (d20 die value = 1): total shown in `var(--pf-blood)`
- Non-d20 rolls: total shown in `var(--pf-gold)` (standard accent)
- Die value squares: 24x24px, `var(--card)` background, `var(--border)` border, 4px radius, mono text centered

### RollHistoryPanel

- Placement: collapsible panel appended below TurnControls in the center column of CombatPage (below the `<TurnControls />` component in the bottom ResizablePanel)
- Default state: collapsed (chevron-down icon, shows "Roll History" label + roll count badge)
- Expanded state: shows up to 50 most recent rolls (newest first); older rolls truncated silently
- Each history row (collapsed single line):
  - `[timestamp HH:MM] [formula Badge] [die breakdown abbreviated: 3+5] [modifier: +4] [= total in gold mono 600]`
  - Row height: 32px, 8px horizontal padding
  - Hover: `var(--muted)` background
- Panel max height when expanded: 240px, overflow-y scroll with `.scrollbar-thin` class
- Header row: "Roll History" label (14px/600) + roll count in muted Badge + spacer + "Clear History" Button (variant: ghost, size: sm)
- "Clear History" button: shows `Trash2` lucide icon + "Clear History" text; click calls `clearRolls()` directly — no confirmation dialog (session-only data, not destructive to persistent state)
- Collapse toggle: ChevronDown/ChevronUp icon button at left of header
- Panel border: `border-t border-border/50` separating from TurnControls above
- Background: `var(--card)` to visually distinguish from panel background

### CombatPage Integration

- RollHistoryPanel appended inside the bottom center ResizablePanel, after `<TurnControls />`:
  ```
  <TurnControls />
  <RollHistoryPanel />  {/* new — collapsed by default */}
  ```
- The bottom ResizablePanel already has `defaultSize={65} minSize={30}` — the collapsed history panel (32px header) fits within this without adjusting panel sizes
- When expanded (240px max), the panel pushes TurnControls and CombatantDetail upward within the scrollable area — no layout changes to the ResizablePanelGroup structure itself
- Sonner `<Toaster />` is already mounted app-wide — RollResultToast uses the existing toaster, no new mount needed

| Element | Copy |
|---------|------|
| Primary CTA (collapse toggle, closed) | "Roll History" |
| Primary CTA (collapse toggle, open) | "Roll History" |

| Empty state heading | "No rolls yet" |
| Empty state body | "Rolls will appear here during the session." |
| Clear button label | "Clear History" |
| Clear button aria-label | "Clear roll history" |
| Toast label fallback (no label) | formula only — no label text |
| Natural 20 toast label suffix | "Critical!" (appended to label in gold) |
| Natural 1 toast label suffix | "Fumble" (appended to label in pf-blood) |
| Error state (roll engine failure) | "Roll failed — try again" (shown in toast variant: destructive) |

Destructive actions in this phase:
- "Clear History" — clears session-only in-memory array; no confirmation dialog required. Data loss is acceptable (session ephemeral). Copy: "Clear History" with `Trash2` icon.
