Export these types:

Export these pure utility functions (testable without Tauri):

1. **`computeHash(content: string): Promise<string>`**
   - Use `window.crypto.subtle.digest('SHA-256', new TextEncoder().encode(content))`
   - Convert ArrayBuffer to hex string: `Array.from(new Uint8Array(hashBuffer)).map(b => b.toString(16).padStart(2, '0')).join('')`

2. **`extractPackName(filePath: string): string`**
   - Split on `/` (paths normalized by Rust glob command)
   - Find the two-segment sequence `packs/pf2e` by searching for index where `parts[i] === 'packs' && parts[i+1] === 'pf2e'`, then return `parts[i+2]`
   - If sequence not found, return `'unknown'`
   - Do NOT use `parts.indexOf('pf2e')` — that matches the root folder name `foundryvtt-pf2e-{sha}` (documented anti-pattern)

3. **`isValidEntity(data: unknown): data is { _id: string; name: string; type: string }`**
   - Type guard returning true only if data is a non-null object with string `_id`, `name`, and `type` properties
   - Per locked decision: do NOT require `system` object

4. **`shouldSkipFile(fileName: string): boolean`**
   - Returns `true` if fileName starts with `_`
   - Per locked decision: filter by filename prefix

Export the main orchestration function:

5. **`syncPacks(onProgress: (progress: SyncProgress) => void): Promise<SyncResult>`**

   Full algorithm (follow plan.txt Phase 2 structure closely):

   **Step 1 - Check release:**
   ```typescript
   import { fetch } from '@tauri-apps/plugin-http';
   const releaseResp = await fetch('https://api.github.com/repos/foundryvtt/pf2e/releases/latest', {
     method: 'GET',
     headers: { 'Accept': 'application/vnd.github.v3+json', 'User-Agent': 'pathbuddy' },
   });
   const release = await releaseResp.json() as { tag_name: string; zipball_url: string };
   ```
   Query syncState via `db.select().from(syncState).limit(1)`. If `lastRelease === release.tag_name`, return early with zeroed SyncResult and `stage: 'done'` progress.

   **Step 2 - Download ZIP:**
   ```typescript
   import { tempDir } from '@tauri-apps/api/path';
   import { invoke } from '@tauri-apps/api/core';
   const tmpPath = await tempDir();
   const zipPath = `${tmpPath}pf2e-release.zip`;
   const extractPath = `${tmpPath}pf2e-extracted`;
   await invoke('download_file', { url: release.zipball_url, destPath: zipPath });
   ```

   **Step 3 - Extract:**
   ```typescript
   await invoke('extract_zip', { zipPath, destPath: extractPath });
   ```

   **Step 4 - Import entities:**
   ```typescript
   const allFiles: string[] = await invoke('glob_json_files', { basePath: extractPath });
   ```
   Filter to only files whose path contains `/packs/pf2e/` segment (some repo files outside packs/ may be .json).

   Process in batches of 500. For each batch, wrap in `BEGIN`/`COMMIT` transaction via `getSqlite()`:
   ```typescript
   const sqlite = await getSqlite();
   await sqlite.execute('BEGIN', []);
   try {
     for (const filePath of batch) { ... }
     await sqlite.execute('COMMIT', []);
   } catch (err) {
     await sqlite.execute('ROLLBACK', []);
     throw err;
   }
   ```

   For each file in batch:
   - Extract fileName from path (last segment after `/`)
   - If `shouldSkipFile(fileName)`, continue
   - `const slug = fileName.replace('.json', '')`
   - `const pack = extractPackName(filePath)`
   - Read content: `const content: string = await invoke('read_text_file', { path: filePath })`
   - Parse JSON in try/catch. On parse failure, `continue` (skip malformed, per locked decision)
   - Validate with `isValidEntity(data)`. If false, `continue`
   - Hash: `const contentHash = await computeHash(content)`
   - Track key: `importedKeys.add(\`${pack}::${slug}\`)`
   - Upsert using Drizzle `onConflictDoUpdate` with `setWhere`:
     ```typescript
     import { sql } from 'drizzle-orm';
     await db.insert(pf2eEntities).values({
       sourceId: data._id,
       pack,
       slug,
       name: data.name,
       entityType: data.type,
       rawData: content,
       contentHash,
       syncedAt: new Date().toISOString(),
     }).onConflictDoUpdate({
       target: [pf2eEntities.pack, pf2eEntities.slug],
       set: {
         sourceId: sql`excluded.source_id`,
         name: sql`excluded.name`,
         entityType: sql`excluded.entity_type`,
         rawData: sql`excluded.raw_data`,
         contentHash: sql`excluded.content_hash`,
         syncedAt: sql`excluded.synced_at`,
       },
       setWhere: sql`${pf2eEntities.contentHash} != excluded.content_hash`,
     });
     ```
   - Report progress every batch: `onProgress({ stage: 'importing', message: \`Importing: ${processed} / ${totalFiles}\`, current: processed, total: totalFiles })`

   **Step 5 - Delete removed entities:**
   ```typescript
   import { eq, and } from 'drizzle-orm';
   const existingEntities = await db.select({ pack: pf2eEntities.pack, slug: pf2eEntities.slug }).from(pf2eEntities);
   const toDelete = existingEntities.filter(e => !importedKeys.has(`${e.pack}::${e.slug}`));
   ```
   Delete in batches of 500 with transaction wrapping. For each entity to delete:
   ```typescript
   await db.delete(pf2eEntities).where(and(eq(pf2eEntities.pack, e.pack), eq(pf2eEntities.slug, e.slug)));
   ```
   Track `result.deleted += toDelete.length`.

   **Step 6 - Update sync_state:**
   ```typescript
   await db.delete(syncState);
   await db.insert(syncState).values({
     lastRelease: release.tag_name,
     lastSyncAt: new Date().toISOString(),
     totalEntities: importedKeys.size,
   });
   ```

   **Step 7 - Cleanup:**
   ```typescript
   await invoke('remove_dir', { path: extractPath });
   await invoke('remove_file', { path: zipPath });
   ```

   Wrap the entire function body in try/catch. On error in steps 2-4 cleanup should still attempt to run (remove temp files). Re-throw the error after cleanup.

   Use `result.added` for newly inserted entities (no prior record for that pack::slug). Since Drizzle upsert doesn't distinguish insert vs update at the SQL level, track the count as `importedKeys.size` at the end and set `result.added = importedKeys.size - (existingEntities.length - toDelete.length)` to approximate new entities. Or simply report `importedKeys.size` as total synced.
