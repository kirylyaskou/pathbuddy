## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. `gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (e.g. `p-2`, `gap-2`) |
| md | 16px | Default element spacing (e.g. `p-4`, `gap-4`) |
| lg | 24px | Section padding (e.g. `p-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Filter row height: 24px (`h-6`) — matches existing ItemFilterPanel pattern
- Search input height: 32px (`h-8`) — matches existing CreatureSearchSidebar
- Filter toggle button height: 28px (`h-7`) — compact, below search input

Source: existing patterns in `CreatureSearchSidebar.tsx` and `ItemFilterPanel.tsx`

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label / meta | 10px (`text-[10px]`) | 400 (regular) | 1.4 |
| Section heading | 12px (`text-xs`) | 600 (semibold) | 1.2 |
| Display / name | 14px (`text-sm`) | 600 (semibold) | 1.5 |

Notes:
- Section labels use `uppercase tracking-wider` (e.g. "Encounters" header, "Tier" label) — established pattern
- Filter count badge uses 10px mono or sans, weight 400
- All font-size decisions pre-populated from existing component patterns

Source: `SavedEncounterList.tsx`, `CreatureSearchSidebar.tsx`

## Color

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `oklch(0.14 0.025 50)` dark | Page background, scroll areas |
| Secondary (30%) | `--secondary` `oklch(0.22 0.035 50)` dark | Cards, sidebar panels, row hover states |
| Accent (10%) | `--primary` `oklch(0.72 0.18 75)` dark (rich gold) | Active filter count badge, selected encounter left-border, active rarity chip |
| Destructive | `--destructive` `oklch(0.55 0.22 25)` dark | Delete encounter button on row hover only |

Accent reserved for:
1. Active filter count badge on the Filters toggle button
2. Selected encounter row left-border (`border-l-2 border-primary`)
3. Active rarity chip border/text when selected
4. "New Encounter" button — `variant="outline"` with primary ring on focus (no filled accent background — ghost/outline style matches header compact)

Rarity chip colors use existing PF2e tokens:
- common: `--pf-rarity-common` (zinc)
- uncommon: `--pf-rarity-uncommon` (gold)
- rare: `--pf-rarity-rare` (purple)
- unique: `--pf-rarity-unique` (pink/magenta)

Source: `globals.css` PF2e color tokens, existing `ItemFilterPanel.tsx` rarity chip pattern

## Component Inventory

### ENC-01 — SavedEncounterList header button

| Property | Spec |
|----------|------|
| Component | `<Button>` from `@/shared/ui/button` |
| Variant | `ghost` |
| Size | `sm` (not `icon`) |
| Label text | `+ New Encounter` |
| Placement | Same header position — right side of `flex items-center justify-between` header row |
| Height | 28px (`h-7`) |
| Font size | 12px (`text-xs`) |
| Icon | `<Plus className="w-3 h-3 mr-1" />` inline before text |

Rationale: Replaces `size="icon"` `w-5 h-5` ghost button. Text label makes the action discoverable. Header is compact (py-2), so `h-7` keeps it flush without expanding header height. No variant change — ghost keeps it low-visual-weight in the sidebar panel.

### ENC-02 — Collapsible filter panel in CreatureSearchSidebar (creatures tab only)

#### Filter toggle button

| Property | Spec |
|----------|------|
| Component | `<Button variant="ghost" size="sm">` |
| Label | `Filters` (when no active filters) / `Filters (N)` (when N filters active) |
| Icon | `<SlidersHorizontal className="w-3 h-3 mr-1.5" />` |
| Active count badge | Inline text `(N)` in `text-primary font-normal` — not a separate Badge component |
| Placement | Below search input, full width |
| Height | 28px (`h-7`) |
| Collapse mechanism | `useState(false)` — no animation required, simple conditional render |
| Default state | Collapsed (filters hidden) |

#### Filter panel container

| Property | Spec |
|----------|------|
| Container | `div` with `space-y-2 pt-1.5` |
| Border | None — visual separation via spacing only |
| Background | Inherits sidebar background |

#### Level range row

| Property | Spec |
|----------|------|
| Layout | `flex items-center gap-1.5` |
| Label | `Lv` — `text-[10px] uppercase tracking-wider text-muted-foreground` |
| Inputs | Two `<Input type="number">` — `w-14 h-6 text-xs px-1.5` each |
| Separator | `–` text in `text-muted-foreground` |
| Placeholder | `min` / `max` |
| Range | 0–25 (PF2e level range) |

Source: matches `ItemFilterPanel.tsx` level range exactly.

#### Rarity multi-select chips

| Property | Spec |
|----------|------|
| Layout | `flex gap-1` |
| Values | `common`, `uncommon`, `rare`, `unique` |
| Behavior | Multi-select (toggle each independently) — array, not single value |
| Inactive style | `text-muted-foreground border border-border/40 hover:border-border px-2 py-0.5 text-[10px] rounded capitalize font-normal` |
| Active style | Rarity token color + `bg-secondary border-current` |
| Rarity colors | `--pf-rarity-{value}` tokens from globals.css |

Note: CONTEXT.md specifies multi-select (`rarity?: string[]`). ItemFilterPanel uses single-select — for this phase, use multi-select (toggle array).

## Interaction States

### Filter toggle (collapsed → expanded)

| State | Visual |
|-------|--------|
| Collapsed, no active filters | `Filters` ghost button, muted icon |
| Collapsed, N active filters | `Filters (N)` — `(N)` in `text-primary` |
| Expanded | Same button text, content visible below |

### Encounter list

| State | Visual |
|-------|--------|
| No encounters | Empty state — see Copywriting Contract |
| Creating (inline input visible) | `<Input h-7 text-sm>` below header, focused |
| Row selected | Left `border-l-2 border-primary`, `bg-secondary/70 font-semibold` |
| Row hover | Delete icon appears (`opacity-0 group-hover:opacity-100`) |
| Running encounter | Green pulse dot `w-2 h-2 rounded-full bg-green-500 animate-pulse` |
