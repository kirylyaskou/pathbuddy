# Plan 19-02: Slot Pip UI + Override Editor

**Phase:** 19 — Spell Slot Tracking + Custom Override
**Requirements:** SLOT-01, SLOT-02, SLOT-03, CUST-01, CUST-02, CUST-03
**Status:** Complete

## Goal

Slot pip UI (●●○○) in the SpellcastingBlock within CreatureStatBlock, shown only when encounterContext is provided. Inline spell override editor (add/remove spells per encounter).

## Changes

- `entities/creature/ui/CreatureStatBlock.tsx`:
  - SlotPips component — renders clickable filled/empty circles per rank; toggles used_count in SQLite
  - AddSpellRow component — FTS5 search dropdown to add a spell to a rank in encounter context
  - SpellcastingBlock extended with encounterContext prop — loads slot state + overrides on mount, persists changes; hover-reveals X button to remove spells; renders AddSpellRow per rank
  - EncounterContext interface exported
- `entities/creature/index.ts` — export EncounterContext type
- `pages/combat/ui/CombatPage.tsx` — passes encounterContext to CreatureStatBlock when isEncounterBacked && selectedId

## Notes

- Slot/override state loads lazily from SQLite when encounterContext is set
- In bestiary context (no encounterContext), slots show static count, no pips, no override controls
- Remove of default spell creates isRemoved=true override; remove of added spell deletes the override row
