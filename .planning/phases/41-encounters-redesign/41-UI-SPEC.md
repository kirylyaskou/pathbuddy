## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tier badge padding, inline chip padding |
| sm | 8px | Row internal padding, compact element spacing |
| md | 16px | Panel internal padding, section gaps |
| lg | 24px | Panel header height, between-section breaks |
| xl | 32px | Major layout gaps |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used this phase |

Exceptions:
- Tab bar height: 36px (h-9 — matches existing `EncounterTabBar`)
- Row height: 32px minimum (py-1.5 = 6px top+bottom + content) — matches existing creature rows
- Split toggle button: 28px hit target (w-7 h-7) — icon-only button in tab bar
- Drag overlay ghost: inherits card dimensions, opacity 0.8

Source: existing `EncounterTabBar.tsx`, `EncounterCreatureList.tsx` patterns

## Interaction Contract

### Drag-and-Drop (Encounters Page)

| Interaction | Behavior |
|-------------|----------|
| Drag start (creature card) | Cursor changes to grabbing, `DragOverlay` renders ghost at 0.8 opacity |
| Drag start (hazard card) | Same; ghost shows `AlertTriangle` icon + amber left border |
| Hover over right panel | Right panel border changes to `border-dashed border-primary/40`, `bg-primary/5` tint |
| Drop on right panel (encounter selected) | Calls `handleAdd` / `addHazardToDraft`; ghost dissolves; row appears in list |
| Drop on right panel (no encounter selected) | No-op; panel shows "Select an encounter first" tooltip for 1.5s |
| Drop cancelled (esc / outside) | Ghost returns to origin via animate back |
| Click-to-add (fallback) | W/+/E tier buttons on creature cards; click on hazard cards — existing behavior preserved |

### Split Mode (Combat Page)

| Interaction | Behavior |
|-------------|----------|
| Split toggle button (inactive) | Visible in tab bar right side only when openTabs.length >= 2; `Columns2` icon, ghost button style |
| Split toggle button (active) | `bg-primary/20 text-primary` to indicate on state |
| Enter split mode | Center panel divides into 2 equal columns; each shows tab[0] and tab[1] initiative list + detail |
| Exit split mode | Returns to single column; activeTabId stays active |
| Click combatant in left column | Updates left column detail + right stat block |
| Click combatant in right column | Updates right column detail + right stat block (overrides left column's last selection) |
| Stat block (right panel) | Shows last selected combatant from either column, no change to panel layout |

## Visual Rules Summary

1. **Three-panel gap**: Use `ResizableHandle withHandle` between all 3 panels — consistent with existing combat page pattern.
2. **Drop target state**: Dashed border + primary/5 tint on the entire right panel div; border-radius 0 (full panel, no rounding on the target wrapper).
3. **Drag ghost**: Match source card dimensions exactly. For creature cards: white/card bg at 0.8 opacity, no shadow. For hazard cards: amber left border preserved in ghost.
4. **Split column divider**: A single 1px vertical `border-r border-border/50` between the two columns in split mode — no resizable handle (equal 50/50 fixed split).
5. **Split toggle button placement**: Flush right in the tab bar, separated from the "+" add-tab button by a `border-l border-border/30` vertical divider. Button is 28px square (w-7 h-7).
6. **Hazard visual treatment in saved encounter list**: Identical to `EncounterCreatureList` amber pattern — amber left border, amber-950 bg, AlertTriangle icon — carried over into persisted encounter combatants view.
7. **No new color tokens**: All colors use existing `globals.css` tokens. No new CSS variables introduced in this phase.
