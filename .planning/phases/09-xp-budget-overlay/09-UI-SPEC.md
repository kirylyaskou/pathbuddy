## XP Overlay Layout Contract

### Structure

The XP overlay is a two-row block inserted **below** the existing sticky toolbar in CombatTracker.vue. It sits above the creature list and is always visible (including empty state).

```
┌─ Sticky toolbar (existing) ─────────────────────────────────────────────────┐
│ Combat Tracker         [Round 1]  [New Round]  [Next Creature]               │
└──────────────────────────────────────────────────────────────────────────────┘
┌─ XP Overlay (new, XpBudgetBar.vue) ─────────────────────────────────────────┐
│ Party  Lvl [─][4][+]  Size [─][4][+]  [x] PWOL      [120 XP]  [Severe ●]   │
├──────────────────────────────────────────────────────────────────────────────┤
│ ████████████████████░░░░░░░░░░░░░░░░  (budget bar, 4px tall)                │
└──────────────────────────────────────────────────────────────────────────────┘
```

**Row 1 — Controls + badges:**
- Left cluster: "Party" label (`text-xs text-stone-400`) + level stepper + size stepper + PWOL checkbox
- Right cluster: XP badge + threat pill
- Layout: `flex items-center justify-between px-6 py-2 bg-charcoal-900 border-b border-charcoal-700`

**Row 2 — Budget bar:**
- Full-width horizontal bar, 4px tall (`h-1`)
- Track: `w-full bg-charcoal-600`
- Fill: `bg-{threat-color}` width set via inline `style="width: {fillPercent}%"` where `fillPercent = min(totalXp / moderateThreshold * 100, 100)`
- Fill anchored to moderate threshold (100% fill = moderate budget). Overflow clamps at 100% — extreme encounters show full bar in the extreme color.
- No animation (CSS transition is acceptable: `transition-all duration-300 ease-out`)
- Layout: `w-full bg-charcoal-600 h-1`

### Party Config Controls

Level stepper:
- Label: "Lvl" (`text-xs text-stone-400`)
- Decrement button: `–` (`text-stone-400 hover:text-stone-200`, `w-5 h-5 flex items-center justify-center bg-charcoal-700 border border-charcoal-600 rounded-l text-xs`)
- Value display: `w-8 text-center text-sm font-bold text-stone-200 bg-charcoal-800 border-t border-b border-charcoal-600` — read-only display (not an `<input>`, avoids browser number spinners)
- Increment button: `+` — mirror of decrement with `rounded-r`
- Range: 1–20. Decrement disabled at 1, increment disabled at 20 (`disabled:opacity-40 disabled:cursor-not-allowed`)

Size stepper: identical pattern, range 1–8.

PWOL checkbox:
- `<input type="checkbox">` with `accent-gold` CSS (native checkbox, gold check color)
- Label: "PWOL" (`text-xs text-stone-400 ml-1`)
- Spacing: `ml-3` gap from size stepper to PWOL group

### XP Badge

```
[120 XP]
```

- Container: `px-3 py-1 rounded-md bg-charcoal-600 border border-charcoal-500 flex items-center gap-1`
- Value: `text-sm font-bold text-stone-200` — e.g. "120"
- Unit: `text-xs text-stone-400` — " XP"
- Empty state (no creatures or all out-of-range): render "0 XP" — same structure, same color. No layout shift.
- Out-of-range indicator: if any creature is out of range, append `text-xs text-amber-400 ml-1` with "†" after the badge. Tooltip not used (desktop DM tool, hover-hint acceptable via `title` attribute: "One or more creatures are beyond the level range and contributed 0 XP").

### Threat Pill

```
[Severe ●]
```

- Container: `px-2 py-0.5 rounded-full text-xs font-bold flex items-center gap-1` + threat-tier background/text classes (see Color table above)
- Dot indicator: `w-1.5 h-1.5 rounded-full bg-current` — same color as text, decorative
- Label: title-cased tier name — "Trivial", "Low", "Moderate", "Severe", "Extreme"
- Empty state (0 creatures): render Trivial pill ("Trivial ●") in grey — per CONTEXT.md "show '0 XP' with '—' threat label." Override: use "Trivial" not "—" since the Trivial pill is visually equivalent to "no threat" and avoids a special empty case in the pill component. The XP badge shows "0 XP" which already communicates no-combat context.

## Interaction Contract

### Party Level Stepper

- Click `–`: decrement `partyLevel` by 1. Disabled and `cursor-not-allowed` at minimum (1).
- Click `+`: increment `partyLevel` by 1. Disabled at maximum (20).
- On change: `useEncounterStore.saveConfig()` called — persists to SQLite. `encounterResult` computed reacts to partyLevel change automatically.
- No debounce — SQLite write is fast and each click is discrete.

### Party Size Stepper

- Identical behavior to level stepper. Range 1–8. Default 4.

### PWOL Checkbox

- Toggle: `v-model="encounterStore.pwol"` — triggers `saveConfig()` on change via `watch`.
- Label "PWOL" is clickable (standard `<label for="...">` binding).
- No confirmation required — PWOL is non-destructive and reversible.

### Budget Bar

- Read-only. No click, no expand/collapse.
- Fill width is a computed inline style: `Math.min((totalXp / moderateBudget) * 100, 100).toFixed(1) + '%'`
- Fill color class swaps reactively with threat tier.
- CSS transition: `transition-all duration-300 ease-out` on the fill element.

### No Destructive Actions

This phase introduces no destructive actions. No confirmation dialogs required.
