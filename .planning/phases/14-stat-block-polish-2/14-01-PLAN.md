<action>
In `src/entities/creature/model/mappers.ts`, add three new handlers to `resolveFoundryTokens()` BEFORE the `@Localize` fallback line (currently line 205). Insert after the `@Template` handler block:

```ts
// [[/act slug]] or [[/act slug #id]] → capitalize slug, hyphens to spaces
// [[/act shove]] → "Shove", [[/act stride #stride]] → "Stride"
// Unknown or empty slug → "" (silently drop)
text = text.replace(/\[\[\/act\s+([^#\s\]]*)[^\]]*\]\]/g, (_, slug: string) => {
  if (!slug) return ''
  return slug
    .replace(/-/g, ' ')
    .replace(/\b\w/g, (c: string) => c.toUpperCase())
})

// [[/br expr #label]]{display} → use display text only
// [[/br 2d6+5 #hit]]{2d6+5} → "2d6+5"
text = text.replace(/\[\[\/br\s+[^\]]*\]\]\{([^}]+)\}/g, '$1')

// [[/br expr]] with NO {display} → use expr as-is
// [[/br 1d6]] → "1d6"
text = text.replace(/\[\[\/br\s+([^#\s\]]+)[^\]]*\]\]/g, '$1')

// {Nfeet} or {Nfoot} where N is digits → "N feet"
// {30feet} → "30 feet", {5foot} → "5 feet"
text = text.replace(/\{(\d+)feet?\}/gi, '$1 feet')
```

**IMPORTANT ordering:** The `[[/br ...]]` WITH display handler must come BEFORE the `[[/br ...]]` WITHOUT display handler, to prevent the no-display regex from consuming tokens that have `{display}`.
</action>

```ts
function formatDamage(damageRolls: any): string {
  if (!damageRolls) return ''

  return entries
```

```ts
function formatDamage(damageRolls: any): { formula: string; type: string }[] {
  if (!damageRolls) return []
  const entries = Object.values(damageRolls) as any[]
  return entries.map((d: any) => ({
    formula: (d.damage || d.formula || '?').trim(),
    type: (d.damageType || d.type || '').trim(),
  }))
}
```

The return type changes from `string` to `{ formula: string; type: string }[]`. The strike mapper at line 62 already calls `formatDamage(item.system?.damageRolls)` — the type change is picked up automatically since the Strike interface now expects an array.
