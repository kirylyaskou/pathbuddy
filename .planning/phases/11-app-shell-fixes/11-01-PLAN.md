2. Add message cycling state and effect inside the `SplashScreen` component:

```tsx
const [msgIndex, setMsgIndex] = useState(() =>
  Math.floor(Math.random() * LOADING_MESSAGES.length)
)
const [msgVisible, setMsgVisible] = useState(true)

useEffect(() => {
  if (status !== 'init' && status !== 'migrating') return
  const interval = setInterval(() => {
    setMsgVisible(false)
    setTimeout(() => {
      setMsgIndex((i) => (i + 1) % LOADING_MESSAGES.length)
      setMsgVisible(true)
    }, 400)
  }, 3000)
  return () => clearInterval(interval)
}, [status])
```

The `msgIndex` starts at a random position so repeated launches feel varied. The `useEffect` cleanup clears the interval when status changes to 'ready' or 'error', preventing unmounted state updates.
