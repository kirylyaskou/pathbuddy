## Overview

Refactor спеллкастинг-модуля + три PF2e-correctness фикса:
- Cantrip trait overrides Foundry `level.value` в catalog (Phase 77)
- castType-aware UI: prepared/innate как consumable copies, spontaneous как pool, focus без per-rank pips (Phase 78)
- Heightening preview в SpellSearchDialog + persistence (Phase 79, migration 0038)
- `use-spellcasting` → facade + 6 sub-hooks (Phase 80)
- Cast-rank через `@item.level` на encounter_combatant_effects (Phase 81, migration 0039)
- FSD migration: `SpellcastingBlock` → `features/spellcasting/` (Phase 82)
- Innate frequency parsing — at-will / N-per-day (Phase 83, migration 0040)

Итого: 3 миграции (0038, 0039, 0040), ~34 коммита.

## Key Decisions

- **Inline-plans вместо per-phase SUMMARY/VERIFICATION.md** — милевый контекст хранился в STATE.md. Причина: шесть взаимозависимых фаз с общим субъектом, оверхед per-phase артефактов не оправдан. Trade-off: затруднил retroactive audit. Для v1.7.0+ — вернуть полный GSD pipeline.
- **castType dispatcher** (Phase 78) — вместо монолитного SpellcastingEditor. Причина: prepared/innate/spontaneous/focus имеют принципиально разные UX, общий компонент расползался.
- **Facade+sub-hooks** (Phase 80) — 6 сфокусированных хуков по single concern. Причина: 393-строчный хук стал нечитаемым; разделение по DB-таблице / derived state.
- **`@item.level` через per-instance `cast_at_rank`** (Phase 81) — вместо шарадки с heightened_from_rank на spell level. Причина: одно и то же заклинание может быть cast на разных rank в одном encounter.
- **FSD migration только на Phase 82** — сначала стабилизировать логику (77-81), потом двигать. Причина: минимизация merge-конфликтов с параллельной работой.
