# Plan 10-03: Dying/Wounded Cascade + Persistent Damage

<objective>
Add a Dying/Wounded cascade dialog that triggers when a combatant reaches 0 HP, with recovery check support (auto-roll + manual d20 input) via engine `performRecoveryCheck`. Add persistent damage tracking as a condition with a flat-check toast prompt on turn advance (DC 15, auto-roll + manual).
</objective>

   ```tsx
     open: boolean
     onClose: () => void
   ```

   ```tsx
   const [manualRoll, setManualRoll] = useState('')

   ```

4. On mount/open, read current dying/wounded/doomed values from condition-bridge:
   ```tsx
   const conditions = useConditionStore(
     useShallow(s => s.activeConditions.filter(c => c.combatantId === combatantId))
   )
   const dyingValue = conditions.find(c => c.slug === 'dying')?.value ?? 0
   const woundedValue = conditions.find(c => c.slug === 'wounded')?.value ?? 0
   const doomedValue = conditions.find(c => c.slug === 'doomed')?.value ?? 0
   ```

5. If combatant doesn't already have dying, apply dying 1 (+ wounded value per PF2e rules):
   On dialog open, if dying is 0:
   ```tsx
   // PF2e: gain dying 1, increased by wounded value
   const initialDying = 1 + woundedValue
   applyCondition(combatantId, 'dying' as ConditionSlug, initialDying)
   ```

6. Death-prevention scan: check abilities for keywords:
   ```tsx
   const DEATH_PREVENTION_KEYWORDS = ['Ferocity', 'Rejuvenation', 'Negative Healing', 'Undead']
   const hasDeathPrevention = abilities?.some(a =>
     DEATH_PREVENTION_KEYWORDS.some(kw => a.name.includes(kw) || a.description.includes(kw))
   )
   ```
   If found, show a warning banner: "This creature may have death-prevention abilities: {matched names}"

7. Recovery check button:
   ```tsx
   const handleRecoveryCheck = () => {
     const roll = rollMode === 'manual' ? parseInt(manualRoll, 10) : undefined
     if (rollMode === 'manual' && (isNaN(roll!) || roll! < 1 || roll! > 20)) return
     const currentDying = conditions.find(c => c.slug === 'dying')?.value ?? 1
     const currentDoomed = conditions.find(c => c.slug === 'doomed')?.value ?? 0
     const result = performRecoveryCheck(currentDying, currentDoomed, roll)
     setCheckResult(result)
     // Apply result to conditions
     if (result.newDyingValue === -1) {
       setIsDead(true)
     } else if (result.stabilized) {
       removeCondition(combatantId, 'dying' as ConditionSlug)
       // Wounded increases by 1 when stabilizing
       applyCondition(combatantId, 'wounded' as ConditionSlug, woundedValue + 1)
     } else {
       // Update dying value
       applyCondition(combatantId, 'dying' as ConditionSlug, result.newDyingValue)
     }
   }
   ```

8. Dialog layout:
   - Header: "Dying — {combatantName}" with skull icon
   - Death-prevention warning (conditional)
   - Current state: Dying {N}, Wounded {N}, Doomed {N}, DC {10 + dying}
   - Roll mode toggle: [Auto Roll] [Manual d20]
   - Manual input: number 1-20
   - "Roll Recovery Check" button
   - Result display: Roll {N} vs DC {N} → {Outcome}, dying {before} → {after}
   - Dead state: large skull icon, "DEAD" text, "Override (DM Fiat)" button
   - Close button

9. Override button sets dying to 1 and clears dead state (DM fiat).
</action>

<task id="3">
<title>Add persistent damage condition support and flat-check on turn advance</title>
<read_first>
- src/features/combat-tracker/ui/ConditionCombobox.tsx (grouped condition list)
- src/features/combat-tracker/lib/turn-manager.ts (advanceTurn — endTurnConditions call)
- src/features/combat-tracker/lib/condition-bridge.ts (endTurnConditions, getManagerState)
- engine/conditions/conditions.ts (CONDITION_SLUGS — check if persistent-damage exists)
</read_first>
<action>
1. Persistent damage is tracked as a special condition. The engine's CONDITION_SLUGS includes standard PF2e conditions. Persistent damage is NOT a standard condition slug — it's tracked as a custom ActiveCondition.

   In `src/entities/condition/model/types.ts` (or wherever ActiveCondition is defined), ensure the type allows a `formula` field:
   ```ts
   export interface ActiveCondition {
     combatantId: string
     slug: string
     value?: number
     isLocked?: boolean
     grantedBy?: string
     formula?: string  // e.g., "2d6 fire" for persistent damage
   }
   ```

2. Add a "Persistent Damage" entry to ConditionCombobox. Add a new group at the top:
   ```tsx
   { label: 'Persistent Damage', slugs: ['persistent-fire', 'persistent-cold', 'persistent-acid', 'persistent-bleed', 'persistent-electricity', 'persistent-poison'] as ConditionSlug[] }
   ```

   When a persistent damage condition is selected, show a formula input (text field for dice like "2d6") instead of the value stepper. Store the formula in the condition:
   ```tsx
   // Apply persistent damage as a custom condition
   useConditionStore.getState().addCondition({
     combatantId,
     slug: `persistent-${damageType}`,
     value: 1,
     formula: formulaInput,  // e.g., "2d6"
   })
   ```

3. In `turn-manager.ts`, after `endTurnConditions` call in `advanceTurn()`, check for persistent damage conditions on the ending combatant:
   ```tsx
   // Check persistent damage
   const activeConditions = useConditionStore.getState()
     .activeConditions.filter(c => c.combatantId === endingCombatantId && c.slug.startsWith('persistent-'))

   if (activeConditions.length > 0) {
     for (const pc of activeConditions) {
       const damageType = pc.slug.replace('persistent-', '')
       const formula = pc.formula || '?'
       toast(
         `${combatant?.displayName}: ${formula} ${damageType} (persistent). Roll flat-check DC 15 to remove.`,
         {
           action: {
             label: 'Roll DC 15',
             onClick: () => {
               const roll = Math.ceil(Math.random() * 20)
               if (roll >= 15) {
                 // Remove persistent damage condition
                 useConditionStore.getState().removeCondition(endingCombatantId!, pc.slug)
                 toast(`Flat-check: ${roll} — Persistent ${damageType} removed!`)
               } else {
                 toast(`Flat-check: ${roll} — Persistent ${damageType} continues. Apply ${formula} damage.`)
               }
             }
           },
           duration: 10000,
         }
       )
     }
   }
   ```

4. The toast is actionable — clicking "Roll DC 15" auto-rolls. The DM can ignore the toast and manually handle it. The persistent damage amount is NOT auto-subtracted (CONTEXT D-16) — DM enters damage in HpControls manually (they may need to apply IWR to persistent damage).
</action>
