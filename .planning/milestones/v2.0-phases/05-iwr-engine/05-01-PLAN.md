**Interfaces (per CONTEXT.md locked decisions):**
- `DamageInstance`: `{ type: DamageType, amount: number, category?: DamageCategory, critical?: boolean, precision?: boolean, materials?: MaterialEffect[] }`
- `Immunity`: `{ readonly type: ImmunityType, readonly exceptions: DamageType[] }`
- `Weakness`: `{ readonly type: WeaknessType, readonly value: number, readonly exceptions: DamageType[], readonly doubleVs?: DoubleVsCondition[], readonly applyOnce: boolean }`
- `Resistance`: `{ readonly type: ResistanceType, readonly value: number, readonly exceptions: DamageType[] }`
- `IWRApplicationResult`: `{ finalDamage: number, appliedImmunities: Immunity[], appliedWeaknesses: Weakness[], appliedResistances: Resistance[] }`

**Factory Functions:**
- `createImmunity(type: ImmunityType, exceptions?: DamageType[]): Immunity` -- defaults exceptions to `[]`
- `createWeakness(type: WeaknessType, value: number, options?: { exceptions?: DamageType[], doubleVs?: DoubleVsCondition[], applyOnce?: boolean }): Weakness` -- defaults: exceptions `[]`, doubleVs `undefined`, applyOnce `false`
- `createResistance(type: ResistanceType, value: number, exceptions?: DamageType[]): Resistance` -- defaults exceptions to `[]`

**Core Engine -- `applyIWR` function:**

Signature: `applyIWR(instance: DamageInstance, immunities: Immunity[], weaknesses: Weakness[], resistances: Resistance[]): IWRApplicationResult`

Processing order (official PF2e CRB):
1. **Immunities pass** -- for each immunity:
   - Skip if excepted: `immunity.exceptions.includes(instance.type)` or (if materials present) `instance.materials?.some(m => immunity.exceptions.includes(m as unknown as DamageType))`
   - Match check: `immunity.type === instance.type` OR `immunity.type === DamageCategorization.getCategory(instance.type)`
   - Special: if `immunity.type === 'critical-hits'` AND `instance.critical === true` => `adjustedAmount = Math.floor(adjustedAmount / 2)`, push to appliedImmunities
   - Special: if `immunity.type === 'precision'` AND `instance.precision === true` => `adjustedAmount = 0`, push to appliedImmunities
   - Standard: if type/category match found => `adjustedAmount = 0`, push to appliedImmunities
2. **Weaknesses pass** (only if adjustedAmount > 0 after immunities):
   - Filter matching weaknesses: `weakness.type === instance.type` OR `weakness.type === DamageCategorization.getCategory(instance.type)`, excluding excepted
   - Material exception check: if `instance.materials?.some(m => weakness.exceptions.includes(m as unknown as DamageType))` => skip
   - From matching, select highest effective value (using `effectiveWeaknessValue` that checks doubleVs: if `weakness.doubleVs?.some(c => c === 'critical' && instance.critical === true)` => `weakness.value * 2`, else `weakness.value`)
   - Apply: `adjustedAmount += highestEffectiveValue`, push the winning weakness to appliedWeaknesses
3. **Resistances pass** (always applies, even if weakness was applied):
   - Filter matching resistances: same type/category match as weakness, excluding excepted
   - Material exception: if instance has a material in the resistance's exceptions => skip
   - Select highest value from matching
   - Apply: `adjustedAmount = Math.max(0, adjustedAmount - highestValue)`, push to appliedResistances
