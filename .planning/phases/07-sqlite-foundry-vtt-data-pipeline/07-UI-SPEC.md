## Splash Screen Interaction Contract

### States

| State | Trigger | Visual |
|-------|---------|--------|
| `init` | App mount, before DB open | Spinner + "Starting..." label |
| `migrating` | DB open, running SQL migrations | Progress bar indeterminate + stage text |
| `syncing` | Not applicable (sync is manual) | n/a on splash |
| `ready` | All migrations complete | Fade out (150ms opacity transition), router mounts |
| `error` | Any DB/migration error | Error card with message + retry button |

### Progress Bar Behavior

- Indeterminate animation during `init` and `migrating` states (no percentage — duration unknown)
- Use shadcn `Progress` component (`shared/ui/progress.tsx` — already installed)
- Stage text below bar: 14px, `--muted-foreground`, updates per migration stage
- Example stage text values: "Opening database...", "Running migrations...", "Ready"

### Transition to Router

- On `ready`: 150ms CSS opacity fade from 1 to 0, then `display: none` / unmount
- No flash of unstyled content — splash is rendered synchronously before `AppProviders`

### Error State

- Error card centered on screen, same card background as splash container
- Icon: `lucide-react` `AlertCircle`, size 24px, color `--destructive`
- Heading: "Failed to initialize database" (20px, weight 600)
- Body: error message string from caught exception (16px, weight 400, `--muted-foreground`)
- Single button: "Retry Initialization" — primary variant, triggers re-run of `initDatabase()`

## Settings Page — Sync Trigger

The "Sync Foundry VTT Data" control lives on the existing Settings page (already scaffolded). No new page created.

### Control Layout

- Section heading: "Data Source" (20px, weight 600)
- Description body: "Import latest Foundry VTT PF2e data from GitHub. Replaces existing entities." (16px, weight 400, `--muted-foreground`)
- Button: "Sync Foundry VTT Data" — primary variant, `lucide-react` `RefreshCw` icon (16px) left of label
- Sync status line below button: last sync date + entity count, 14px, `--muted-foreground`

### Sync In-Progress State

While sync runs (after button clicked):

- Button: disabled, spinner replaces icon, label changes to "Syncing..."
- Progress area appears below button: `Progress` component (indeterminate → determinate as chunks arrive)
- Stage text: "Downloading ZIP...", "Extracting packs...", "Importing entities (1,247 / 28,076)...", "Building search index..."
- On complete: progress area fades out, status line updates to "Last synced: just now — 28,076 entities"
- On error: `sonner` toast (already installed via `shared/ui/sonner.tsx`): "Sync failed — [error message]. Try again."

### Destructive Action

Sync overwrites all existing entity data. This is not treated as a destructive confirmation action because:
- There is no user-created data in the entities table (all data is from Foundry VTT)
- Re-sync is the intended update mechanism

No confirmation dialog required for sync. Decision basis: D-02 (manual trigger), D-04 (all entities replaced by design).

## Accessibility Contract

- Splash `Progress` component: `role="progressbar"`, `aria-valuenow` bound to percentage (or `aria-valuemin=0 aria-valuemax=100` without `aria-valuenow` during indeterminate)
- Splash stage text: `aria-live="polite"` region so screen readers announce stage changes
- Splash error: `role="alert"` on error container for immediate announcement
- Sync button disabled state: `aria-disabled="true"` while sync is in progress
- Sync progress area: `aria-live="polite"` region for stage text updates
