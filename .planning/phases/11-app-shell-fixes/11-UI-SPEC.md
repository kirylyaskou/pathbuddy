## D20 Spinner — Visual Specification

This is the primary new visual element in Phase 11. Full spec so executor needs no design decisions.

### Layout

```
[full-screen bg-background]
  ↕ flex-col items-center justify-center
    [D20Spinner]         ← ~160px container, perspective: 600px
    [h1 "PathBuddy"]     ← text-[28px] font-semibold text-pf-gold tracking-wide
    [loading message]    ← text-sm text-muted-foreground (fade transition)
    [progress bar]       ← w-48 h-0.5 bg-pf-gold/40 animate-pulse
```

Gap between stack items: `gap-6` (24px). Total stack centered with `flex flex-col items-center gap-6 p-8`.

### D20Spinner Component

- **Container:** `w-40 h-40` (160px), `style={{ perspective: '600px' }}`
- **Spinning wrapper:** `relative w-full h-full`, `style={{ transformStyle: 'preserve-3d', animation: 'spin-d20 8s infinite linear' }}`
- **Faces:** 12 triangle `<div>` elements (approximation — top cap + 5 upper ring at `rotateY(72deg * n)` + 5 lower ring at `rotateY(72deg * n + 36deg)` + bottom cap)
- **Face clip-path:** `polygon(50% 0%, 100% 100%, 0% 100%)`
- **Face background:** `oklch(0.14 0.025 50 / 0.85)` (inline style)
- **Face border:** `1px solid oklch(0.75 0.18 75 / 0.5)` (inline style)
- **Face `translateZ`:** `90px` for all ring faces; `0px` for cap faces with rotateX adjustment

### Keyframe (globals.css — outside any @layer)

```css
@keyframes spin-d20 {
  0%   { transform: rotateX(-20deg) rotateY(0deg); }
  100% { transform: rotateX(-20deg) rotateY(360deg); }
}
```

Animation duration: **8 seconds** — slow and cinematic (D-03). Linear timing, infinite loop.

### Message Cycling

- Interval: **3 seconds** per message
- Fade-out duration: **400ms** (`transition-opacity duration-400`)
- Fade-out fires first (400ms), then index increments, then fade-in
- `aria-live="polite"` on message `<p>` element (preserved from existing component)
- Message list: 25 curated TTRPG-universal messages (see RESEARCH.md code examples)
- Randomize starting index with `Math.floor(Math.random() * MESSAGES.length)` so repeated loads feel varied

### Progress Bar

- Width: `w-48` (192px), height: `h-0.5` (2px)
- Track: `bg-border` with `overflow-hidden rounded-full`
- Fill: `bg-pf-gold/40 animate-pulse w-full`
- Purpose: subtle — the d20 animation is the primary loading indicator (D-08)

### Error State (unchanged visually)

Keep existing error card layout exactly as coded in `SplashScreen.tsx`:
- Card: `bg-card rounded-lg p-8`
- Icon: `AlertCircle` from lucide, `h-6 w-6 text-destructive`
- Heading: "Failed to initialize database" — `text-xl font-semibold text-foreground`
- Body: `{error} — restart the app or contact support.` — `text-base text-muted-foreground`
- CTA: shadcn `<Button>` — label: "Retry Initialization" (preserved from existing)

Source: D-09 "Keep error state functional — existing behavior". Executor must not change the error state visual.

## ScrollArea Fix — Visual Specification

No new visual design. The fix is a behavior change with identical appearance.

### Contract

- Replace `src/shared/ui/scroll-area.tsx` Radix implementation with native `<div className="relative overflow-y-auto scrollbar-thin">`
- Preserve `ScrollArea` and `ScrollBar` named exports (import compatibility for 6 consumers)
- `ScrollBar` export becomes a no-op (`return null`) — no consumer renders it as a visible element
- Add `.scrollbar-thin` utility class to `globals.css` (outside any `@layer` block):

```css
.scrollbar-thin {
  scrollbar-width: thin;
  scrollbar-color: oklch(0.30 0.04 55) transparent;
}
.scrollbar-thin::-webkit-scrollbar { width: 6px; }
.scrollbar-thin::-webkit-scrollbar-track { background: transparent; }
.scrollbar-thin::-webkit-scrollbar-thumb {
  background-color: oklch(0.30 0.04 55);
  border-radius: 3px;
}
```

Scrollbar thumb color (`oklch(0.30 0.04 55)`) matches `--border` dark-mode token — neutral, not distracting.

No changes to the 6 consuming files (`BestiaryPage.tsx`, `BestiarySearchPanel.tsx`, `InitiativeList.tsx`, `CreatureSearchSidebar.tsx`, `EncounterCreatureList.tsx`, and any others).
